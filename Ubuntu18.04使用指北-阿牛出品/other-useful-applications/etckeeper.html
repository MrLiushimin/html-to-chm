etckeeper

etckeeper允许/ etc的内容存储在版本控制系统（VCS）存储库中。它与APT集成，并在安装或升级软件包时自动提交对/ etc的更改 。在版本控制下放置/ etc被认为是行业最佳实践，而etckeeper的目标是使这个过程尽可能轻松。

通过在终端中输入以下内容来 安装etckeeper： 

sudo apt install etckeeper

主配置文件/etc/etckeeper/etckeeper.conf非常简单。主要选项是使用哪个VCS，默认情况下，etckeeper配置为使用Bazaar。在软件包安装期间，存储库会自动初始化（并首次提交）。可以通过输入以下命令来撤消此操作： 

sudo etckeeper uninit

默认情况下，etckeeper将每天提交对/ etc进行的未提交的更改。可以使用AVOID_DAILY_AUTOCOMMITS配置选项禁用此选项。它还将在包安装之前和之后自动提交更改。为了更精确地跟踪更改，建议使用以下方法手动提交更改以及提交消息： 

sudo etckeeper提交“..配置改变的原因......”

使用bzr的VCS命令可以查看日志信息： 

sudo bzr log / etc / passwd

要演示与包管理系统（APT）的集成，请安装postfix： 

sudo apt install postfix

有关etckeeper如何跟踪手动更改的示例，请将新主机添加到/ etc / hosts。使用bzr，您可以看到哪些文件已被修改： 

sudo bzr status / etc / 
modified：
  主机

现在提交更改： 

sudo etckeeper提交“添加新主机”

有关bzr的更多信息，请参阅Bazaar。
