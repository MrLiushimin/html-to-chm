OpenVPN的

如果您不仅需要预共享密钥，OpenVPN 可以轻松设置和使用公钥基础结构（PKI）来使用SSL / TLS证书进行VPN服务器和客户端之间的身份验证和密钥交换。 OpenVPN可以在路由或桥接VPN模式下使用，并且可以配置为使用UDP或TCP。端口号也可以配置，但端口1194是官方端口号。并且它只使用该单个端口进行所有通信。VPN客户端实现几乎适用于所有Linux发行版，OS X，Windows和基于OpenWRT的WLAN路由器。

服务器安装

要在终端中安装openvpn，请输入： 

sudo apt install openvpn easy-rsa

公钥基础设施设置

构建OpenVPN配置的第一步是建立PKI（公钥基础结构）。PKI包括：

  • 一个单独的证书（也称为公钥）和服务器以及每个客户端的私钥，以及

  • 主证书颁发机构（CA）证书和密钥，用于签署每个服务器和客户端证书。

OpenVPN支持基于证书的双向身份验证，这意味着客户端必须对服务器证书进行身份验证，并且服务器必须在建立相互信任之前对客户端证书进行身份验证。

服务器和客户端都将通过首先验证提供的证书是否由主证书颁发机构（CA）签名，然后通过测试现已验证的证书头中的信息（例如证书公用名或证书类型（客户端）来验证另一个或服务器）。

证书颁发机构设置

要设置自己的证书颁发机构（CA）并为OpenVPN服务器和多个客户端生成证书和密钥，请首先将easy-rsa目录复制到/ etc / openvpn。这将确保在更新程序包时不会丢失对脚本的任何更改。从终端更改到用户root并： 

mkdir / etc / openvpn / easy-rsa / 
cp -r / usr / share / easy-rsa / * / etc / openvpn / easy-rsa /

接下来，编辑/ etc / openvpn / easy-rsa / vars将以下内容调整到您的环境： 

出口KEY_COUNTRY =“美国”
export KEY_PROVINCE =“NC”
export KEY_CITY =“Winston-Salem”
export KEY_ORG =“示例公司”
导出KEY_EMAIL =“steve@example.com”
导出KEY_CN = MyVPN
export KEY_ALTNAMES = AltMyVPN
导出KEY_NAME = MyVPN
导出KEY_OU = MyVPN

输入以下内容以生成主证书颁发机构（CA）证书和密钥： 

cd / etc / openvpn / easy-rsa / 
source vars 
./clean-all 
./build-ca

服务器证书

接下来，我们将为服务器生成证书和私钥： 

./build-key-server myservername

与上一步一样，大多数参数都可以默认。另外两个查询需要肯定响应，“签署证书？[y / n]”和“1个证书请求中的1个已经认证，提交？[y / n]”。

必须为OpenVPN服务器生成Diffie Hellman参数： 

./build-dh

所有证书和密钥都已在子目录keys /中生成。通常的做法是将它们复制到/ etc / openvpn /： 

cd keys / 
cp myservername.crt myservername.key ca.crt dh2048.pem / etc / openvpn /

客户证书

VPN客户端还需要一个证书来向服务器验证自身。通常，您为每个客户端创建不同的证书。要创建证书，请在用户root用户的终端中输入以下内容： 

cd / etc / openvpn / easy-rsa / 
source vars 
./build-key client1

使用安全方法将以下文件复制到客户端：

  • /etc/openvpn/ca.crt

  • /etc/openvpn/easy-rsa/keys/client1.crt

  • /etc/openvpn/easy-rsa/keys/client1.key

由于仅在客户端计算机上需要客户端证书和密钥，因此应将其从服务器中删除。

＃把net.ipv4.ip_forward = 1

sudo sysctl -p /etc/sysctl.conf

第一次遇到麻烦

如果上述方法不适合您，请检查以下内容：

  • 检查你的日志，例如journalctl --identifier ovpn-server（对于server.conf）

  • 检查是否在client.conf和server.conf中正确指定了密钥文件名。

  • 客户端可以连接到服务器机器吗？也许防火墙会阻止访问？检查服务器上的日志。

  • 客户端和服务器必须使用相同的协议和端口，例如UDP端口1194，请参阅端口和proto配置选项

  • 客户端和服务器必须使用相同的压缩配置，请参阅comp-lzo配置选项

  • 客户端和服务器必须使用有关桥接模式与路由模式的相同配置，请参阅server vs server-bridge config选项

高级配置

服务器上的高级路由VPN配置

以上是一个非常简单的工作VPN。客户端可以通过加密隧道访问VPN服务器计算机上的服务。如果要访问更多服务器或其他网络中的任何内容，请将某些路由推送到客户端。例如，如果您公司的网络可以汇总到网络192.168.0.0/16，您可以将此路由推送到客户端。但是您还必须更改回路的路由 - 您的服务器需要知道到VPN客户端网络的路由。

或者您可以将默认网关推送到所有客户端，以便首先将所有互联网流量发送到VPN网关，然后通过公司防火墙将其发送到互联网。本节向您展示了一些可能的选项。

将路由推送到客户端以允许它到达服务器后面的其他私有子网。请记住，这些私有子网还需要知道将OpenVPN客户端地址池（10.8.0.0/24）路由回OpenVPN服务器。

服务器上的高级桥接VPN配置

可以为路由或桥接VPN模式设置OpenVPN。有时这也被称为OSI第2层与第3层VPN。在桥接VPN中，所有第2层帧（例如所有以太网帧）都被发送到VPN伙伴，并且在路由VPN中，只有第3层数据包被发送到VPN伙伴。在桥接模式中，包括传统上LAN本地的流量（如本地网络广播，DHCP请求，ARP请求等）的所有流量都被发送到VPN伙伴，而在路由模式下，这将被过滤。

首先，在客户端上安装openvpn： 

sudo apt install openvpn

然后，在配置服务器并将客户端证书复制到/ etc / openvpn /目录的情况下，通过复制示例来创建客户端配置文件。在客户端计算机上的终端中输入： 

sudo cp /usr/share/doc/openvpn/examples/sample-config-files/client.conf / etc / openvpn

现在编辑/etc/openvpn/client.conf更改以下选项： 

开发
; dev tun
ca ca.crt
cert client1.crt
key client1.key

最后，重启openvpn： 

sudo systemctl重启openvpn @ client

您现在应该可以通过VPN连接到远程LAN。
