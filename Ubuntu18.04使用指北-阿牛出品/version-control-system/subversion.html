颠覆

Subversion是一个开源版本控制系统。使用Subversion，您可以记录源文件和文档的历史记录。它随着时间的推移管理文件和目录。将文件树放入中央存储库。存储库很像普通的文件服务器，除了它记住对文件和目录所做的每一次更改。

  • 服务器配置
  • 访问方法

安装

要使用HTTP协议访问Subversion存储库，您必须安装和配置Web服务器。Apache2被证明可以与Subversion一起使用。请参阅Apache2部分中的HTTP子部分来安装和配置Apache2。要使用HTTPS协议访问Subversion存储库，必须在Apache 2 Web服务器中安装和配置数字证书。请参阅Apache2部分中的HTTPS子部分以安装和配置数字证书。

要安装Subversion，请从终端提示符处运行以下命令： 

sudo apt install subversion apache2 libapache2-svn

服务器配置

此步骤假定您已在系统上安装了上述软件包。本节介绍如何创建Subversion存储库并访问项目。

创建Subversion存储库

可以使用终端提示符中的以下命令创建Subversion存储库：

svnadmin create / path / to / repos / project

导入文件

创建存储库后，可以 将文件导入存储库。要导入目录，请在终端提示符下输入以下内容： 

svn import / path / to / import / directory file：/// path / to / repos / project

访问方法

Subversion存储库可以通过许多不同的方法访问（检出） - 在本地磁盘或通过各种网络协议。但是，存储库位置始终是URL。该表描述了不同的URL方案如何映射到可用的访问方法。

在本节中，我们将了解如何为所有这些访问方法配置Subversion。在这里，我们涵盖了基础知识。有关更高级的用法详细信息，请参阅svn书籍。

直接存储库访问（文件：//）

这是所有访问方法中最简单的。它不需要运行任何Subversion服务器进程。此访问方法用于从同一台计算机访问Subversion。在终端提示符下输入的命令语法如下：

svn co file：/// path / to / repos / project

要么

svn co file：// localhost / path / to / repos / project

如果未指定主机名，则有三个正斜杠（///） - 两个用于协议（在本例中为文件）加上路径中的前导斜杠。如果指定主机名，则必须使用两个正斜杠（//）。

存储库权限取决于文件系统权限。如果用户具有读/写权限，则可以从存储库签出并提交。

通过WebDAV协议访问（http：//）

要通过WebDAV协议访问Subversion存储库，您必须配置Apache 2 Web服务器。在/etc/apache2/sites-available/000-default.conf中的<VirtualHost>和</ VirtualHost>元素 或其他VirtualHost文件之间添加以下代码段 ： 

 <Location / svn>
  DAV svn
  SVNParentPath / path / to / repos
  AuthType Basic
  AuthName“您的存储库名称”
  AuthUserFile / etc / subversion / passwd
  需要有效用户
 </位置>

上面的配置代码段假定 使用svnadmin命令在/ path / to / repos目录下创建Subversion存储库，并且HTTP用户对文件具有足够的访问权限（参见下文）。可以使用http：// hostname / svn / repos_name url 访问它们。

像上面那样更改apache配置需要使用以下命令重新加载服务 

    sudo systemctl reload apache2.service

要通过HTTP将文件导入或提交到Subversion存储库，存储库应归HTTP用户所有。在Ubuntu系统中，HTTP用户是 www-data。要更改存储库文件的所有权，请在终端提示符处输入以下命令： 

    sudo chown -R www-data：www-data / path / to / repos

通过将存储库的所有权更改为 www-data，您将无法通过以除www-data之外的任何用户身份运行svn import file：///命令 来将文件导入或提交到存储库中。

接下来，您必须创建包含用户身份验证详细信息的/ etc / subversion / passwd文件。要创建文件，请在命令提示符处发出以下命令（将创建文件并添加第一个用户）： 

sudo htpasswd -c / etc / subversion / passwd user_name

要添加其他用户，请省略“-c”选项，因为此选项将替换旧文件。而是使用此表单： 

sudo htpasswd / etc / subversion / passwd user_name

此命令将提示您输入密码。输入密码后，即会添加用户。现在，要访问存储库，您可以运行以下命令：

svn co http：// servername / svn

密码以纯文本形式传输。如果您担心密码窥探，建议您使用SSL加密。有关详细信息，请参阅下一节。

使用SSL加密通过WebDAV协议访问（https：//）

使用SSL加密（https：//）通过WebDAV协议访问Subversion存储库与http：//类似，不同之处在于您必须在Apache2 Web服务器中安装和配置数字证书。要在Subversion中使用SSL，请将上述Apache2配置添加到/etc/apache2/sites-available/default-ssl.conf。有关使用SSL设置Apache2的更多信息，请参阅HTTPS配置。

您可以安装签名机构颁发的数字证书。或者，您可以安装自己的自签名证书。

此步骤假定您已在Apache 2 Web服务器中安装并配置了数字证书。现在，要访问Subversion存储库，请参阅上面的部分！除协议外，访问方法完全相同。您必须使用https：//访问Subversion存储库。

通过自定义协议访问（svn：//）

创建Subversion存储库后，您可以配置访问控制。您可以编辑 /path/to/repos/project/conf/svnserve.conf文件以配置访问控制。例如，要设置身份验证，您可以取消注释配置文件中的以下行：

＃ [一般]
#password-db = passwd

取消注释上述行后，您可以在passwd文件中维护用户列表。因此，在同一目录中编辑文件passwd 并添加新用户。语法如下： 

用户名=密码

有关更多详细信息，请参阅该文件。

现在，要通过svn：//自定义协议从同一台计算机或不同的计算机访问Subversion，您可以使用svnserve命令运行svnserver。语法如下： 

$ svnserve -d --foreground -r / path / to / repos
＃-d  - 守护进程模式
＃--foreground  - 在前台运行（对调试很有用）
＃-r  - 要服务的目录的根目录

有关更多用法详情，请参阅：
$ svnserve --help

运行此命令后，Subversion开始侦听默认端口（3690）。要访问项目存储库，必须从终端提示符运行以下命令：

svn co svn：// hostname / project project --username user_name

根据服务器配置，它会提示输入密码。一旦通过身份验证，它就会从Subversion存储库中检出代码。要将项目存储库与本地副本同步，可以运行update子命令。在终端提示符下输入的命令语法如下： 

cd project_dir; svn更新

有关使用每个Subversion子命令的更多详细信息，请参阅手册。例如，要了解有关co（checkout）命令的更多信息，请从终端提示符处运行以下命令：

svn co help

使用SSH加密通过自定义协议访问（svn + ssh：//）

配置和服务器进程与svn：//方法中的相同。有关详细信息，请参阅上一节。此步骤假定您已执行上述步骤并使用svnserve 命令启动Subversion服务器。

还假设ssh服务器正在该机器上运行，并且它允许传入连接。要确认，请尝试使用ssh登录该计算机。如果你能登录，一切都很完美。如果您无法登录，请在继续之前解决。

svn + ssh：//协议用于使用SSL加密访问Subversion存储库。使用此方法加密数据传输。要访问项目存储库（例如，使用签出），必须使用以下命令语法： 

    svn co svn + ssh：// ssh_username @ hostname / path / to / repos / project

您必须使用完整路径（/ path / to / repos / project）使用此访问方法访问Subversion存储库。

根据服务器配置，它会提示输入密码。您必须输入用于通过ssh登录的密码。经过身份验证后，它会检出Subversion存储库中的代码。
