访问Samba共享

另一个用于Samba的是集成到现有的Windows网络中。一旦成为Active Directory域的一部分，Samba就可以为AD用户提供文件和打印服务。

加入AD域的最简单方法是使用Same-open。有关详细说明，请参阅 Same Open文档。

一旦成为Active Directory域的一部分，请在终端提示符中输入以下命令： 

sudo apt install samba cifs-utils smbclient

接下来，编辑/etc/samba/smb.conf更改： 

   workgroup =示例
   ...
   安全=广告
   realm = EXAMPLE.COM
   ...
   idmap backend = lwopen
   idmap uid = 50-9999999999
   idmap gid = 50-9999999999

重新启动samba以使新设置生效： 

sudo systemctl restart smbd.service nmbd.service

您现在应该能够从Windows客户端访问任何Samba共享。但是，请确保为相应的AD用户或组授予对共享目录的访问权限。有关详细信息，请参阅保护文件和打印服务器。

访问Windows共享

既然Samba服务器是Active Directory域的一部分，您可以访问任何Windows服务器共享：

  • 要挂载Windows文件共享，请在终端提示符中输入以下内容： 

    mount.cifs //fs01.example.com/share mount_point

    也可以访问不属于AD域的计算机上的共享，但需要提供用户名和密码。

  • 要在引导期间安装共享，请在/ etc / fstab中放置一个条目，例如： 

    //192.168.0.5/share / mnt / windows cifs auto，username = steve，password = secret，rw 0 0

  • 从Windows服务器复制文件的另一种方法是使用smbclient实用程序。列出Windows共享中的文件： 

    smbclient //fs01.example.com/share -k -c“ls”

  • 要从共享中复制文件，请输入： 

    smbclient //fs01.example.com/share -k -c“get file.txt”

    这会将file.txt复制到当前目录中。

  • 并将文件复制到共享： 

    smbclient //fs01.example.com/share -k -c“put / etc / hosts hosts”

    这会将/ etc / hosts复制到//fs01.example.com/share/hosts。

  • 上面使用 的-c选项允许您一次执行smbclient命令。这对脚本和次要文件操作很有用。要输入smb：\> 提示符，可以执行普通文件和目录命令的FTP提示，只需执行： 

    smbclient //fs01.example.com/share -k

替换的所有实例fs01.example.com/share，//192.168.0.5/share， 用户名=史蒂夫，密码=秘密，并file.txt的与你的服务器的IP，主机名，共享名，文件名，和实际的用户名和具有共享权限的密码。
