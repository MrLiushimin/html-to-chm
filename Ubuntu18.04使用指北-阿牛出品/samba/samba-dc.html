作为域控制器

可以将Samba服务器配置为显示为Windows NT4样式的域控制器。此配置的一个主要优点是能够集中用户和计算机凭据。Samba还可以使用多个后端来存储用户信息。

主域控制器

本节介绍如何使用默认的smbpasswd后端将Samba配置为主域控制器（PDC）。

  1. 首先，通过在终端提示符中输入以下内容，安装Samba和libpam-winbind来同步用户帐户： 

    sudo apt安装samba libpam-winbind

  2. 接下来，通过编辑/etc/samba/smb.conf来配置Samba 。在安全模式应设置为用户和工作组应与您的组织： 

       workgroup =示例
   ...
   security = user

  3. 在评论的“域名”部分中添加或取消注释以下内容（最后一行已拆分以适合此文档的格式）： 

       域登录=是
   登录路径= \\％N \％U \ profile
   登录驱动器= H：
   登录home = \\％N \％U
   登录脚本= logon.cmd
   添加机器脚本= sudo / usr / sbin / useradd -N -g机器-c机器-d
         / var / lib / samba -s / bin / false％u

    如果您不想使用漫游配置文件，请登录登录主页 和登录路径选项。

    • 域登录：提供netlogon服务，使Samba充当域控制器。

    • 登录路径：将用户的Windows配置文件放入其主目录。也可以配置[profiles]共享将所有配置文件放在单个目录下。

    • logon drive：指定主目录本地路径。

    • logon home：指定主目录位置。

    • 登录脚本：确定用户登录后要在本地运行的脚本。脚本需要放在[netlogon]共享中。

    • 添加机器脚本：一个脚本，它将自动创建工作站加入域所需的 机器信任帐户。

      在此示例中，需要使用addgroup实用程序创建计算机组 ，有关详细信息，请参阅添加和删​​除用户。

  4. 取消注释[homes]共享以允许 映射登录主页： 

    [家庭]
   comment =主页目录
   可浏览=否
   只读=否
   创建掩码= 0700
   目录掩码= 0700
   有效用户=％S

  5. 配置为域控制器时，需要配置[netlogon]共享。要启用共享，请取消注释： 

    [NETLOGON]
   comment =网络登录服务
   path = / srv / samba / netlogon
   客人ok =是的
   只读=是
   分享模式=没有

    原始的netlogon共享路径是/ home / samba / netlogon，但根据文件系统层次结构标准（FHS）， / srv是系统提供的特定于站点的数据的正确位置。

  6. 现在创建netlogon目录，以及一个空的（现在） logon.cmd脚本文件： 

    sudo mkdir -p / srv / samba / netlogon 
sudo touch /srv/samba/netlogon/logon.cmd

    您可以在logon.cmd中输入任何常规Windows登录脚本命令来自定义客户端的环境。

  7. 重新启动Samba以启用新域控制器： 

    sudo systemctl restart smbd.service nmbd.service

  8. 最后，还需要一些其他命令来设置适当的权限。

    随着根被默认禁用，为了加入工作站到域，系统组需要被映射到Windows 域管理员组。使用net实用程序，从终端输入： 

    sudo net groupmap add ntgroup =“Domain Admins”unixgroup = sysadmin rid = 512 type = d

    将系统管理员更改为您喜欢的任何组。此外，用于加入域的用户需要是sysadmin组的成员，以及系统管理员组的成员。该管理员组允许 sudo的使用。

    如果用户尚未拥有Samba凭据，则可以使用smbpasswd实用程序添加它们，并相应地更改sysadmin用户名： 

    sudo smbpasswd -a sysadmin

    此外，需要向Domain Admins组明确提供权限，以允许 添加计算机脚本（和其他管理功能）工作。这是通过执行： 

    net rpc rights grant -U sysadmin“EXAMPLE \ Domain Admins”SeMachineAccountPrivilege \
SePrintOperatorPrivilege SeAddUsersPrivilege SeDiskOperatorPrivilege \
SeRemoteShutdownPrivilege

  9. 您现在应该可以将Windows客户端加入域中，方法与将它们加入到Windows服务器上运行的NT4域的方式相同。

备份域控制器

使用网络上的主域控制器（PDC），最好也有备份域控制器（BDC）。这将允许客户端在PDC变得不可用时进行身份验证。

将Samba配置为BDC时，您需要一种方法来同步PDC的帐户信息。有多种方法可以完成此scp，rsync或使用LDAP作为passdb后端。

使用LDAP是同步帐户信息的最强大方法，因为两个域控制器都可以实时使用相同的信息。但是，对于少量用户和计算机帐户，设置LDAP服务器可能过于复杂。有关详细信息，请参阅Samba和LDAP。

  1. 首先，安装samba和libpam-winbind。从终端输入： 

    sudo apt安装samba libpam-winbind

  2. 现在，编辑/etc/samba/smb.conf并在[global]中取消注释以下内容： 

       workgroup =示例
   ...
   security = user

  3. 在评论的域中取消注释或添加： 

       域登录=是
   域名主=不

  4. 确保用户有权读取/ var / lib / samba中的文件。例如，要允许 admin组中的用户scp文件，请输入： 

    sudo chgrp -R admin / var / lib / samba

  5. 接下来，同步用户帐户，使用scp 从PDC 复制/ var / lib / samba目录： 

    sudo scp -r username @ pdc：/ var / lib / samba / var / lib

    将用户名替换为有效的用户名，将pdc替换为实际PDC的主机名或IP地址。

  6. 最后，重启samba： 

    sudo systemctl restart smbd.service nmbd.service

您可以通过停止PDC上的Samba守护程序，然后尝试登录到加入域的Windows客户端来测试备份域控制器是否正常工作。

另外要记住的是，如果您已将登录主页选项配置为PDC上的目录，并且PDC变得不可用，则对用户的主驱动器的访问也将不可用。因此，最好将登录主页配置为驻留在PDC和BDC的单独文件服务器上。
