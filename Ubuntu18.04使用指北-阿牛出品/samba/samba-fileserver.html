文件服务器

联网Ubuntu和Windows计算机的最常用方法之一是将Samba配置为文件服务器。本节介绍如何设置Samba服务器以与Windows客户端共享文件。

服务器将配置为与网络上的任何客户端共享文件，而不会提示输入密码。如果您的环境需要更严格的访问控制，请参阅保护文件和打印服务器。

  • 组态

安装

第一步是安装samba包。从终端提示输入： 

sudo apt安装samba

这里的所有都是它的; 您现在可以配置Samba以共享文件。

组态

主Samba配置文件位于/etc/samba/smb.conf中。默认配置文件具有大量注释，以便记录各种配置指令。

并非所有可用选项都包含在默认配置文件中。有关更多详细信息，请参见smb.conf 手册页或 Samba HOWTO Collection。

  1. 首先，在/etc/samba/smb.conf的[global]部分中 编辑以下键/值对： 

       workgroup =示例
   ...
   security = user

    该安全参数在[global]部分更远了下来，并且在默认情况下评论。此外，更改示例以更好地匹配您的环境。

  2. 在文件底部创建一个新部分，或取消注释其中一个示例，以便共享目录： 

    [分享]
    comment = Ubuntu文件服务器共享
    path = / srv / samba / share
    可浏览=是的
    客人ok =是的
    只读=否
    创建掩码= 0755

    • 评论：分享的简短描述。调整以满足您的需求。

    • path：要共享的目录的路径。

      此示例使用/ srv / samba / sharename，因为根据 文件系统层次结构标准（FHS）， / srv 是应该提供特定于站点的数据的位置。从技术上讲，只要权限正确，Samba共享就可以放在文件系统的任何位置，但建议遵守标准。

    • browsable：允许Windows客户端使用Windows资源管理器浏览共享目录 。

    • guest ok：允许客户端连接到共享而无需提供密码。

    • 只读：确定共享是只读还是授予写权限。仅当值为no时才允许写入权限，如此示例中所示。如果值为yes，则只读访问共享。

    • create mask：确定新文件创建时的权限。

  3. 现在已配置Samba，需要创建目录并更改权限。从终端输入： 

    sudo mkdir -p / srv / samba / share 
sudo chown nobody：nogroup / srv / samba / share /

    该-p开关告诉MKDIR创建整个目录树，如果它不存在。

  4. 最后，重新启动samba服务以启用新配置： 

    sudo systemctl restart smbd.service nmbd.service

再一次，上述配置可以访问本地网络上的任何客户端。有关更安全的配置，请参阅保护文件和打印服务器。

从Windows客户端，您现在应该能够浏览到Ubuntu文件服务器并查看共享目录。如果您的客户端未自动显示您的共享，请尝试在Windows资源管理器窗口中通过其IP地址（例如\\ 192.168.1.1）访问您的服务器。要检查一切是否正常，请尝试从Windows创建目录。

要创建其他共享，只需在/etc/samba/smb.conf中创建新的[dir]部分 ，然后重新启动Samba。只需确保您要共享的目录确实存在且权限正确。

名为“[share]”的文件共享和路径/ srv / samba / share只是示例。调整共享和路径名称以适合您的环境。在文件系统上的目录之后命名共享是个好主意。另一个例子是[qa]的共享名称，其路径为/ srv / samba / qa。
