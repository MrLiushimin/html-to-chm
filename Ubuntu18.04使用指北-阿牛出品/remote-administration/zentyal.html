Zentyal

Zentyal是一个Linux小型企业服务器，可以配置为网关，基础架构管理器，统一威胁管理器，办公服务器，统一通信服务器或它们的组合。Zentyal管理的所有网络服务都紧密集成，可以自动执行大多数任务。这样可以节省时间并有助于避免网络配置和管理中的错误。 Zentyal是开源的，在GNU通用公共许可证（GPL）下发布，运行在Ubuntu GNU / Linux之上。

Zentyal由一系列包（通常每个模块一个）组成，提供Web界面来配置不同的服务器或服务。配置存储在键值Redis数据库中，但用户，组和域相关配置在OpenLDAP上 。通过Web界面配置任何可用参数时，将使用模块提供的配置模板覆盖最终配置文件。使用Zentyal的主要优势是统一的图形用户界面，可配置所有网络服务以及它们之间的高度，开箱即用的集成。

Zentyal根据最新的Ubuntu LTS版本每年发布一次主要稳定版本。

安装

如果要创建新用户以访问Zentyal Web界面，请运行： 

sudo adduser用户名sudo

将Zentyal存储库添加到存储库列表： 

sudo add-apt-repository“deb http://archive.zentyal.org/zentyal 3.5 main extra”

从Zentyal导入公钥： 

sudo apt-key adv --keyserver keyserver.ubuntu.com --recv-keys 10E239FF 
wget -q http://keys.zentyal.org/zentyal-4.2-archive.asc -O- | sudo apt-key add  -

更新您的软件包并安装Zentyal： 

sudo apt update 
sudo apt install zentyal
在安装过程中，将要求您设置root MySQL密码并确认端口443。

第一步

允许属于sudo组的任何系统帐户登录 Zentyal Web界面。安装Ubuntu Server时创建的用户默认属于sudo组。

要访问Zentyal Web界面，请将浏览器指向https：// localhost /或远程服务器的IP地址。由于 Zentyal创建自己的自签名SSL证书，您必须在浏览器上接受安全例外。使用用于登录服务器的相同用户名和密码登录。

登录后，您将看到服务器的概述。只需单击它们，然后单击“安装”即可安装各个模块，如防病毒或防火墙。选择Gateway或Infrastructure等服务器角色可以同时安装多个模块。

也可以通过命令行安装模块： 

sudo apt install <zentyal-module>
请参阅下面的可用模块列表。

要启用模块，请转到仪表板，然后单击“模块状态”。单击模块的复选框，然后单击保存更改。

要配置已安装模块的任何功能，请单击左侧菜单中的不同部分。进行任何更改时，右上角会出现一个红色的“保存更改”按钮。

如果您需要自定义任何配置文件或运行某些操作（脚本或命令）来配置Zentyal上不可用的功能 ，请将自定义配置文件模板放在/ etc / zentyal / stubs / <module> /和/ etc /上的挂钩上zentyal /钩/ <模块> <动作>。在这里阅读有关存根和钩子的更多信息。

模块

Zentyal 2.3可以在Ubuntu 18.04 Universe存储库中找到。可用的模块是：

  • zentyal-core＆zentyal-common：Zentyal接口的核心 和框架的通用库。还包括日志和事件模块，这些模块为管理员提供了查看日志和从中生成事件的界面。

  • zentyal-network：管理网络的配置。从具有多个Internet连接的接口（支持静态IP，DHCP，VLAN，网桥或PPPoE）到多个网关，负载平衡和高级路由，静态路由或动态DNS。

  • zentyal-objects＆zentyal-services：为网络地址（例如LAN而不是192.168.1.0/24）和名为服务的端口（例如HTTP而不是80 / TCP）提供抽象级别。

  • zentyal-firewall：配置iptables 规则以阻止禁止连接，NAT和端口重定向。

  • zentyal-ntp：安装NTP守护程序以保持服务器准时，并允许网络客户端将其时钟与服务器同步。

  • zentyal-dhcp：配置ISC DHCP服务器，支持网络范围，静态租约和其他高级选项，如NTP，WINS，动态DNS更新和PXE网络启动。

  • zentyal-dns：将ISC Bind9 DNS服务器带入您的服务器，以便将本地查询作为转发器或配置域的权威服务器进行缓存。允许配置A，CNAME，MX，NS，TXT和SRV记录。

  • zentyal-ca：在Zentyal中集成了证书颁发机构的管理，因此用户可以使用证书对服务进行身份验证，例如OpenVPN。

  • zentyal-openvpn：允许使用OpenVPN配置多个VPN服务器和客户端，并使用Quagga进行动态路由配置。

  • zentyal-users：提供在OpenLDAP上配置和管理用户和组的界面。Zentyal上的其他服务通过LDAP进行身份验证，具有集中用户和组管理。还可以从Microsoft Active Directory域同步用户，密码和组。

  • zentyal-squid：由于缓存功能和内容过滤，配置Squid和 Dansguardian以加快浏览速度。

  • zentyal-samba：允许Samba配置并与现有LDAP集成。您可以从同一界面定义密码策略，创建共享资源和分配权限。

  • zentyal-printers：将CUPS与Samba集成， 不仅可以配置打印机，还可以根据LDAP用户和组授予他们权限。

在Ubuntu Universe存储库中不存在，但在 Zentyal Team PPA上， 您将找到以下其他模块：

  • zentyal-antivirus：将ClamAV 防病毒软件与代理，文件共享或mailfilter等其他模块集成在一起。

  • zentyal-asterisk：配置Asterisk 以提供基于LDAP的身份验证的简单PBX。

  • zentyal-bwmonitor：允许监视LAN客户端的带宽使用情况。

  • zentyal-captiveportal：将强制门户与防火墙和LDAP用户和组集成。

  • zentyal-ebackup：允许使用流行的duplicity备份工具对服务器进行预定备份。

  • zentyal-ftp：使用基于LDAP的身份验证配置FTP服务器。

  • zentyal-ids：集成了网络入侵检测系统。

  • zentyal-ipsec：允许使用OpenSwan配置IPsec隧道 。

  • zentyal-jabber：将ejabberd XMPP服务器与LDAP用户和组集成。

  • zentyal-thinclients：一个LTSP基于瘦客户机溶液。

  • zentyal-mail：一个完整​​的邮件堆栈，包括带有LDAP后端的Postfix 和Dovecot。

  • zentyal-mailfilter：使用邮件堆栈配置amavisd以过滤垃圾邮件和附加的病毒。

  • zentyal显示器：集成collectd 监控服务器的性能和运行的服务。

  • zentyal-pptp：配置PPTP VPN服务器。

  • zentyal-radius：将FreeRADIUS与LDAP用户和组集成。

  • zentyal-software：管理已安装的Zentyal模块和系统更新的简单界面 。

  • zentyal-trafficshaping：配置流量限制规则以进行带宽限制并改善延迟。

  • zentyal-usercorner：允许用户使用Web浏览器编辑自己的LDAP属性。

  • zentyal-virt：基于libvirt创建和管理虚拟机的简单界面。

  • zentyal-webmail：允许使用流行的Roundcube webmail 访问您的邮件 。

  • zentyal-webserver：将Apache Web 服务器配置为托管计算机上的不同站点。

  • zentyal-zarafa：将Zarafa 群件套件与Zentyal邮件堆栈和LDAP 集成在一起。
