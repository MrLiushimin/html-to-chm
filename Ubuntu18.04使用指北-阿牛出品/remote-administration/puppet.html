木偶

Puppet是一个跨平台框架，使系统管理员可以使用代码执行常见任务。代码可以执行各种任务，从安装新软件到检查文件权限或更新用户帐户。 Puppet不仅在系统的初始安装期间非常出色，而且在整个系统的整个生命周期中都很出色。在大多数情况下， puppet将用于客户端/服务器配置。

本节将介绍如何在客户端/服务器配置中安装和配置Puppet。这个简单的例子将演示如何使用Puppet安装Apache。

预配置

在配置puppet之前，您可能希望为puppet.example.com添加DNS CNAME记录 ，其中example.com是您的域。默认情况下， Puppet客户端将puppet.example.com的DNS检查为puppet服务器名称或 Puppet Master。有关更多DNS详细信息，请参阅域名服务（DNS）。

如果您不想使用DNS，则可以向服务器和客户端/ etc / hosts文件添加条目。例如，在 Puppet服务器的/ etc / hosts文件中添加： 

127.0.0.1 localhost.localdomain localhost puppet
192.168.1.17 puppetclient.example.com puppetclient

在每个Puppet客户端上，为服务器添加一个条目： 

192.168.1.16 puppetmaster.example.com puppetmaster puppet

将上面的示例IP地址和域名替换为您的实际服务器和客户端地址和域名。

安装

要安装Puppet，请在服务器上的终端中输入： 

sudo apt安装puppetmaster

在客户端计算机或计算机上，输入： 

sudo apt安装木偶

组态

为apache2类创建文件夹路径： 

  sudo mkdir -p / etc / puppet / modules / apache2 / manifests

现在为apache2设置一些资源。创建一个 包含以下内容的文件/etc/puppet/modules/apache2/manifests/init.pp： 

class apache2 {
  package {'apache2'：
    确保=>已安装，
  }

  service {'apache2'：
    ensure => true，
    enable => true，
    require => Package ['apache2']，
  }
}

接下来，使用以下命令创建节点文件/etc/puppet/code/environments/production/manifests/site.pp： 

node'puppetclient.example.com'{
   包括apache2
}

将puppetclient.example.com替换为您实际的Puppet客户端主机名。

这个简单的Puppet服务器的最后一步是重启守护进程： 

sudo systemctl重启puppetmaster.service

现在一切都在Puppet服务器上配置，是时候配置客户端了。

首先，配置Puppet代理守护程序以启动。编辑/ etc / default / puppet，将START更改 为yes： 

START = YES

然后启动服务： 

sudo systemctl start puppet.service

查看客户端证书指纹 

sudo木偶代理 - 指纹

返回Puppet服务器，查看挂起的证书签名请求： 

sudo木偶证书清单

在Puppet服务器上，验证客户端的指纹并签署puppetclient的证书： 

sudo puppet cert sign puppetclient.example.com

在Puppet客户端上，在前台手动运行puppet代理。严格来说，这一步并非必要，但它是测试和调试puppet服务的最佳方法。 

sudo木偶代理 - 测试

检查两台主机上的/ var / log / syslog是否存在配置错误。如果一切顺利，apache2 包及其依赖项将安装在Puppet客户端上。

这个例子非常简单，并没有突出Puppet的许多功能和优点。欲了解更多信息，请参阅参考资料。
