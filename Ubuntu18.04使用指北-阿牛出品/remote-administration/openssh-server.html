OpenSSH服务器

  • 介绍
  • 安装
  • 组态
  • SSH密钥
  • 参考

介绍

“Ubuntu服务器指南”的这一部分介绍了一组功能强大的工具，用于远程控制和传输名为OpenSSH的联网计算机之间的数据。您还将了解OpenSSH服务器应用程序可能的一些配置设置以及如何在Ubuntu系统上更改它们。

OpenSSH是Secure Shell（SSH）协议系列工具的免费版本，用于远程控制或在计算机之间传输文件。用于完成这些功能的传统工具（例如 telnet或rcp）是不安全的，并且在使用时以明文形式传输用户密码。OpenSSH提供服务器守护程序和客户端工具，以促进安全，加密的远程控制和文件传输操作，有效地取代传统工具。

OpenSSH服务器组件sshd持续监听来自任何客户端工具的客户端连接。发生连接请求时，sshd会根据连接的客户端工具的类型设置正确的连接。例如，如果远程计算机与ssh客户端应用程序连接，则OpenSSH服务器在身份验证后设置远程控制会话。如果远程用户使用scp连接到OpenSSH服务器，则OpenSSH服务器守护程序会在身份验证后在服务器和客户端之间启动文件的安全副本。OpenSSH可以使用许多身份验证方法，包括普通密码，公钥和Kerberos 门票。

安装

安装OpenSSH客户端和服务器应用程序很简单。要在Ubuntu系统上安装OpenSSH客户端应用程序，请在终端提示符处使用此命令： 

sudo apt安装openssh-client

要安装OpenSSH服务器应用程序和相关支持文件，请在终端提示符处使用此命令： 

sudo apt安装openssh-server

在OpenSSH的服务器软件包，也可以选择在服务器版的安装过程中安装。

组态

您可以通过编辑文件 / etc / ssh / sshd_config来配置OpenSSH服务器应用程序sshd的默认行为 。有关此文件中使用的配置指令的信息，您可以使用在终端提示符下发出的以下命令查看相应的手册页： 

man sshd_config

sshd配置文件中 有许多指令控制通信设置和身份验证模式等内容。以下是可以通过编辑/ etc / ssh / sshd_config文件来更改的配置指令示例。

在编辑配置文件之前，您应该复制原始文件并保护其不被写入，这样您就可以将原始设置作为参考，并在必要时重复使用。

复制/ etc / ssh / sshd_config文件并使用在终端提示符下发出的以下命令保护它不被写入： 

sudo cp / etc / ssh / sshd_config /etc/ssh/sshd_config.original 
sudo chmod aw /etc/ssh/sshd_config.original

以下是您可能更改的配置指令的示例：

  • 要将OpenSSH设置为侦听TCP端口2222而不是默认TCP端口22，请更改Port指令：

    港口2222

  • 要让sshd允许基于公钥的登录凭据，只需添加或修改该行：

    PubkeyAuthentication是的

    如果该行已存在，则确保未注释掉该行。

  • 要使您的OpenSSH服务器将/etc/issue.net文件的内容显示 为登录前横幅，只需添加或修改该行：

    横幅/etc/issue.net

    在/ etc / ssh / sshd_config文件中。

更改/ etc / ssh / sshd_config文件后，保存文件，然后在终端提示符下使用以下命令重新启动sshd服务器应用程序以实现更改： 

sudo systemctl restart sshd.service

sshd的 许多其他配置指令可用于更改服务器应用程序的行为以满足您的需要。但是，请注意，如果您访问服务器的唯一方法是 ssh，并且在通过 / etc / ssh / sshd_config文件配置sshd时出错 ，则可能会在重新启动时发现您已被锁定在服务器之外。此外，如果提供了错误的配置指令，sshd服务器可能会拒绝启动，因此在远程服务器上编辑此文件时要格外小心。

SSH密钥

SSH 密钥允许在两台主机之间进行身份验证，而无需密码。SSH密钥认证使用两个密钥，一个私有密钥和公共密钥。

要生成密钥，请在终端提示符下输入： 

ssh-keygen -t rsa

这将使用RSA算法生成密钥。在此过程中，系统将提示您输入密码。只需在提示创建密钥时按Enter键。

默认情况下，公共密钥保存在文件的〜/ .ssh / id_rsa.pub，而 的〜/ .ssh / id_rsa是私有密钥。现在将id_rsa.pub文件复制到远程主机，并通过输入以下内容将其附加到〜/ .ssh / authorized_keys： 

ssh-copy-id username @ remotehost

最后，仔细检查authorized_keys文件的权限，只有经过身份验证的用户才具有读写权限。如果权限不正确，请通过以下方式更改它们： 

chmod 600 .ssh / authorized_keys

您现在应该能够在不提示输入密码的情况下SSH到主机。
