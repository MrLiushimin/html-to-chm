邮差

Mailman是一个用于管理电子邮件讨论和电子通讯列表的开源程序。许多开源邮件列表（包括所有Ubuntu邮件列表）都使用Mailman作为他们的邮件列表软件。它功能强大且易于安装和维护。

安装

Mailman为管理员和用户提供Web界面，使用外部邮件服务器发送和接收电子邮件。它与以下邮件服务器完美配合：

  • 后缀

  • 进出口

  • 发邮件

  • Qmail的

我们将看到如何安装和配置Mailman，Apache Web服务器以及Postfix或Exim邮件服务器。如果您希望使用其他邮件服务器安装Mailman，请参阅参考资料部分。

您只需要安装一个邮件服务器，Postfix是默认的Ubuntu邮件传输代理。

的Apache2

要安装apache2，请参阅安装以获取详细信息。

后缀

有关安装和配置Postfix的说明，请参阅Postfix

的exim4

要安装Exim4，请参阅Exim4。

安装exim4后，配置文件将存储在/ etc / exim4目录中。在Ubuntu中，默认情况下，exim4配置文件分为不同的文件。您可以通过更改/etc/exim4/update-exim4.conf文件中的以下变量来更改此行为： 

dc_use_split_config = '真'

邮差

要安装Mailman，请在终端提示符处运行以下命令： 

sudo apt安装mailman

它将安装文件复制到 / var / lib / mailman目录中。它将CGI脚本安装在/ usr / lib / cgi-bin / mailman 目录中。它创建列表 linux用户。它创建了列表 linux组。邮递员进程将由该用户拥有。

组态

本节假定您已成功安装 mailman， apache2和 postfix或exim4。现在你只需要配置它们。

的Apache2

一个示例Apache配置文件随Mailman一起提供，位于/etc/mailman/apache.conf中。为了让Apache使用配置文件，需要将其复制到/ etc / apache2 / sites-available： 

sudo cp /etc/mailman/apache.conf /etc/apache2/sites-available/mailman.conf

这将为Mailman管理站点设置新的Apache VirtualHost。现在启用新配置并重新启动Apache： 

sudo a2ensite mailman.conf 
sudo systemctl restart apache2.service

Mailman使用apache2来呈现其CGI脚本。邮件CGI脚本安装在/ usr / lib / cgi-bin / mailman 目录中。因此，邮递员网址将是http：// hostname / cgi-bin / mailman /。如果要更改此行为，可以对 /etc/apache2/sites-available/mailman.conf文件进行更改。

后缀

对于Postfix集成，我们将域lists.example.com与邮件列表相关联。请将lists.example.com替换为您选择的域名。

您可以使用postconf命令将必要的配置添加到/etc/postfix/main.cf： 

sudo postconf -e'relay_domains = lists.example.com'sudo 
postconf -e'transport_maps = hash：/ etc / postfix / transport'sudo 
postconf -e'mailman_destination_recipient_limit = 1'

在/etc/postfix/master.cf中，仔细检查您是否有以下传输： 

mailman unix  -  nn  -   -  pipe
  flags = FR user = list argv = / usr / lib / mailman / bin / postfix-to-mailman.py
  $ {nexthop} $ {user}

当邮件传递到列表时， 它会调用postfix-to-mailman.py脚本。

使用传输映射将域lists.example.com与Mailman传输相关联。编辑文件/ etc / postfix / transport： 

lists.example.com mailman：

现在让Postfix通过在终端提示符下输入以下内容来构建传输映射： 

sudo postmap -v / etc / postfix / transport

然后重新启动Postfix以启用新配置： 

sudo systemctl重启postfix.service

的exim4

安装Exim4后，可以使用终端提示符中的以下命令启动Exim服务器： 

sudo systemctl start exim4.service

为了使邮件员能够使用Exim4，您需要配置Exim4。如前所述，默认情况下，Exim4使用不同类型的多个配置文件。有关详细信息，请参阅Exim网站。要运行mailman，我们应该将新配置文件添加到以下配置类型：

  • 主要

  • 运输

  • 路由器

Exim通过对所有这些迷你配置文件进行排序来创建主配置文件。因此，这些配置文件的顺序非常重要。

主要

属于主类型的所有配置文件都存储在 /etc/exim4/conf.d/main/目录中。您可以将以下内容添加到名为04_exim4-config_mailman的新文件中：

#Lomeman安装的主目录 - 也就是Mailman的前缀
＃ 目录。
＃在Ubuntu上这应该是“/ var / lib / mailman”

MM_HOME =的/ var / lib中/邮差

运输

属于传输类型的所有配置文件都存储在/etc/exim4/conf.d/transport/目录中。您可以将以下内容添加到名为 40_exim4-config_mailman的新文件中： 

  mailman_transport：
   driver = pipe
   command = MM_WRAP \
               '$ {if def：local_part_suffix \
                     {$ {sg {$ local_part_suffix} { - （\\ w +）（\\ +。*）？} {\ $ 1}}} \
                     {post}}'\
               $ local_part
    current_directory = MM_HOME
    home_directory = MM_HOME
    user = MM_UID
    group = MM_GID

路由器

属于路由器类型的所有配置文件都存储在 /etc/exim4/conf.d/router/目录中。您可以将以下内容添加到名为101_exim4-config_mailman的新文件中： 

  mailman_router：
   司机=接受
   require_files = MM_HOME / lists / $ local_part / config.pck
   local_part_suffix_optional
   local_part_suffix = -bounces：-bounces + *：\
                       -confirm + *：-join：-leave：\
                       -owner：-request：-admin
   transport = mailman_transport

主服务器和传输配置文件的顺序可以是任何顺序。但是，路由器配置文件的顺序必须相同。此特定文件必须出现在 200_exim4-config_primary文件之前。这两个配置文件包含相同类型的信息。第一个文件优先。有关详细信息，请参阅参考部分。

邮差

安装mailman后，您可以使用以下命令运行它： 

sudo systemctl start mailman.service

安装mailman后，您应该创建默认邮件列表。运行以下命令以创建邮件列表：

管理

我们假设您有默认安装。mailman cgi脚本仍在 / usr / lib / cgi-bin / mailman /目录中。Mailman提供基于Web的管理工具。要访问此页面，请将浏览器指向以下URL：

HTTP：//主机名/ cgi-bin目录/邮差/管理

默认邮件列表mailman将显示在此屏幕中。如果单击邮件列表名称，它将询问您的验证密码。如果输入正确的密码，则可以更改此邮件列表的管理设置。您可以使用命令行实用程序（/ usr / sbin / newlist）创建新的邮件列表。或者，您可以使用Web界面创建新的邮件列表。

用户

Mailman为用户提供基于Web的界面。要访问此页面，请将浏览器指向以下URL：

HTTP：//主机名/ cgi-bin目录/邮差/列表信息

默认邮件列表mailman将显示在此屏幕中。如果单击邮件列表名称，它将显示订阅表单。您可以输入您的电子邮件地址，姓名（可选）和密码进行订阅。系统会向您发送电子邮件邀请。您可以按照电子邮件中的说明进行订阅。
