的exim4

Exim4是剑桥大学开发的另一种消息传输代理（MTA），用于连接到Internet的Unix系统。进出口可以安装到位的sendmail的，虽然配置 进出口是到完全不同的sendmail。

安装

要安装exim4，请运行以下命令： 

sudo apt install exim4

组态

要配置Exim4，请运行以下命令： 

sudo dpkg-reconfigure exim4-config

将显示用户界面。用户界面允许您配置许多参数。例如，在 Exim4中，配置文件分为多个文件。如果您希望将它们放在一个文件中，则可以在此用户界面中进行相应配置。

您在用户界面中配置的所有参数都存储在 /etc/exim4/update-exim4.conf.conf文件中。如果要重新配置，请重新运行配置向导或使用您喜欢的编辑器手动编辑此文件。配置完成后，可以运行以下命令生成主配置文件： 

sudo update-exim4.conf

生成主配置文件，并将其存储在 /var/lib/exim4/config.autogenerated中。

您无需 手动编辑主配置文件 /var/lib/exim4/config.autogenerated。每次运行update-exim4.conf时都会自动 更新

您可以运行以下命令来启动 Exim4守护程序。 

sudo systemctl start exim4.service

SMTP验证

本节介绍如何将Exim4配置为使用带有TLS和SASL的SMTP-AUTH。

第一步是创建用于TLS的证书。在终端提示中输入以下内容： 

sudo / usr / share / doc / exim4-base / examples / exim-gencert

现在需要通过编辑/etc/exim4/conf.d/main/03_exim4-config_tlsoptions为TLS配置Exim4， 添加以下内容： 

MAIN_TLS_ENABLE =是的

接下来，您需要配置Exim4以使用saslauthd 进行身份验证。编辑/etc/exim4/conf.d/auth/30_exim4-config_examples并取消注释plain_saslauthd_server和login_saslauthd_server部分： 

 plain_saslauthd_server：
   driver = plaintext
   public_name = PLAIN
   server_condition = $ {if saslauthd {{$ auth2} {$ auth3}} {1} {0}}
   server_set_id = $ auth2
   server_prompts =：
   .ifndef AUTH_SERVER_ALLOW_NOTLS_PASSWORDS
   server_advertise_condition = $ {if eq {$ tls_cipher} {} {} {*}}
   。万一
＃
 login_saslauthd_server：
   driver = plaintext
   public_name = LOGIN
   server_prompts =“Username ::：Password ::”
   ＃不要通过未加密的连接发送系统密码
   server_condition = $ {if saslauthd {{$ auth1} {$ auth2}} {1} {0}}
   server_set_id = $ auth1
   .ifndef AUTH_SERVER_ALLOW_NOTLS_PASSWORDS
   server_advertise_condition = $ {if eq {$ tls_cipher} {} {} {*}}
   。万一

此外，为了使外部邮件客户端能够连接到新的exim服务器，需要使用以下命令将新用户添加到exim中。 

sudo / usr / share / doc / exim4-base / examples / exim-adduser

用户应使用以下命令保护新的exim密码文件。

sudo chown root：Debian-exim / etc / exim4 / passwd 
sudo chmod 640 / etc / exim4 / passwd

最后，更新Exim4配置并重新启动服务： 

sudo update-exim4.conf 
sudo systemctl restart exim4.service

配置SASL

本节提供有关配置saslauthd以便为Exim4提供身份验证的详细信息。

第一步是安装sasl2-bin软件包。在终端提示符下输入以下内容： 

sudo apt install sasl2-bin

要配置saslauthd，请编辑/ etc / default / saslauthd配置文件并将START = no设置为： 

START = YES

接下来，Debian-exim用户需要成为sasl组的一部分，以便Exim4使用saslauthd服务： 

sudo adduser Debian-exim sasl

现在开始saslauthd服务： 

sudo systemctl start saslauthd.service

Exim4现在使用TLS和SASL身份验证配置SMTP-AUTH。
