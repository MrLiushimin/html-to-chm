Dovecot服务器

Dovecot是一个邮件传递代理，主要考虑安全性。它支持主要的邮箱格式：mbox或Maildir。本节介绍如何将其设置为imap或pop3服务器。

安装

要安装具有常见pop3和imap函数的基本Dovecot服务器，请在命令提示符中运行以下命令： 

sudo apt install dovecot-imapd dovecot-pop3d

还有其他各种Dovecot模块，如dovecot-sieve（邮件过滤），dovecot-solr（全文搜索），......

组态

要配置达夫科特，你可以编辑文件 /etc/dovecot/dovecot.conf并在其列入configfiles /etc/dovecot/conf.d/。默认情况下，所有已安装的协议都将通过/etc/dovecot/dovecot.conf中的include指令启用。 

！include_try /usr/share/dovecot/protocols.d/*.protocol

IMAPS和POP3S更安全，因为它们使用SSL加密进行连接。一个基本的自签名ssl证书由包ssl-cert自动设置，并由dovecot在/etc/dovecot/conf.d/10-ssl.conf中使用。

默认配置mbox格式，如果需要，您也可以使用maildir。有关这方面的更多信息，请参阅/etc/dovecot/conf.d//10-mail.conf上的评论 。在Dovecot网站上讨论了其他优点和详细信息。

您应配置邮件传输代理（MTA）以将传入邮件传输到所选类型的邮箱（如果它与您配置的邮箱不同）。

配置dovecot后，重新启动Dovecot守护程序以测试您的设置： 

sudo systemctl restart dovecot.service

如果您启用了imap或pop3，您还可以尝试使用命令telnet localhost pop3或 telnet localhost imap2登录。如果您看到以下内容，则安装成功： 

bhuvan @ rainbow：〜$ telnet localhost pop3
试试127.0.0.1 ......
连接到localhost.localdomain。
逃脱角色是'^]'。
+好的Dovecot准备好了。

Dovecot SSL配置

Dovecot现在自动配置为使用SSL。它使用包ssl-cert，它提供自签名证书。如果要设置自定义证书，可以编辑文件/etc/dovecot/conf.d/10-ssl.conf并修改以下行（有关详细信息，请参阅证书。）： 

ssl_cert = </etc/dovecot/private/dovecot.pem
ssl_key = </etc/dovecot/private/dovecot.key

您可以从证书颁发机构获取SSL证书，也可以创建自签名SSL证书。请参阅证书 有关如何创建自签名证书的详细信息。创建证书后，您将拥有一个密钥文件和一个证书文件，您希望在上面显示的配置中知道该文件。

电子邮件服务器的防火墙配置

要从另一台计算机访问邮件服务器，必须将防火墙配置为允许在必要端口上连接到服务器。

  • IMAP - 143

  • IMAPS - 993

  • POP3 - 110

  • POP3S - 995
