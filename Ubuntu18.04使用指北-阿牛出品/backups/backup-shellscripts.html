Shell Scripts

备份系统的最简单方法之一是使用shell脚本。例如，脚本可用于配置要备份的目录，并将这些目录作为参数传递给tar实用程序，后者将创建存档文件。然后可以将存档文件移动或复制到另一个位置。也可以在远程文件系统（如NFS安装）上创建存档。

该焦油实用程序创建一个档案文件了许多文件或目录。 tar 还可以通过压缩实用程序过滤文件，从而减小归档文件的大小。

简单的Shell脚本

以下shell脚本使用tar在远程安装的NFS文件系统上创建存档文件。存档文件名使用其他命令行实用程序确定。 

＃！/斌/庆典
####################################
＃
＃备份到NFS安装脚本。
＃
####################################

＃备份什么 
backup_files =“/ home / var / spool / mail / etc / root / boot / opt”

＃备份到哪里。
DEST = “/ MNT /备份”

＃创建存档文件名。
day = $（日期+％A）
hostname = $（hostname -s）
ARCHIVE_FILE = “$ hostname- $ day.tgz”

＃打印开始状态消息。
echo“将$ backup_files备份到$ dest / $ archive_file”
日期
回声

＃使用tar备份文件。
tar czf $ dest / $ archive_file $ backup_files

＃打印结束状态消息。
回声
echo“备份完成”
日期

＃$ dest中文件的长列表，用于检查文件大小。
ls -lh $ dest

  • $ backup_files：一个变量，列出了您要备份的目录。该列表应根据您的需求进行定制。

  • $ day：一个持有星期几的变量（星期一，星期二，星期三等）。这用于为一周中的每一天创建存档文件，提供七天的备份历史记录。还有其他方法可以实现此目的，包括使用日期实用程序。

  • $ hostname：包含系统短主机名的变量。使用存档文件名中的主机名可以选择将来自多个系统的每日存档文件放在同一目录中。

  • $ archive_file：完整存档文件名。

  • $ dest：存档文件的目标。需要创建目录，并在此情况下 在执行备份脚本之前安装。有关使用NFS的详细信息，请参阅网络文件系统（NFS）。

  • 状态消息：使用echo 实用程序打印到控制台的可选消息。

  • tar czf $ dest / $ archive_file $ backup_files：用于创建存档文件的tar命令。

    • c：创建存档。

    • z：通过压缩存档的gzip实用程序过滤存档。

    • f：输出到存档文件。否则，tar输出将被发送到STDOUT。

  • ls -lh $ dest： optional语句以-h 目标目录的可读格式打印-l long列表。这对于存档文件的快速文件大小检查很有用。此检查不应取代测试存档文件。

这是备份shell脚本的一个简单示例; 但是有很多选项可以包含在这样的脚本中。有关提供更深入的shell脚本信息的资源的链接，请参见参考。

执行脚本

从终端执行

执行上述备份脚本的最简单方法是将内容复制并粘贴到文件中。 例如，backup.sh。该文件必须可执行： 

chmod u + x backup.sh

然后从终端提示： 

sudo ./backup.sh

这是测试脚本以确保一切按预期工作的好方法。

用cron执行

该cron的实用程序可用于自动化脚本执行。所述的cron守护进程允许脚本或命令的执行，在指定的时间和日期。

cron是通过crontab文件中的条目配置的。 crontab文件分为以下字段： 

#mh dom mon dow命令

  • m：命令执行时间，介于0和59之间。

  • h：命令执行的小时，介于0和23之间。

  • dom：命令执行的月份日期。

  • mon：命令执行的月份，介于1和12之间。

  • dow：命令执行的星期几，介于0和7之间。可以使用0或7指定星期日，两个值都有效。

  • command：要执行的命令。

要在crontab文件中添加或更改条目，应使用crontab -e命令。此外，可以使用crontab -l命令查看crontab文件的内容。

使用cron 执行上面列出的backup.sh脚本。在终端提示符下输入以下内容： 

sudo crontab -e

使用sudo和crontab -e命令可以编辑root用户的crontab。如果要备份只有root用户有权访问的目录，则必须执行此操作。

将以下条目添加到crontab文件： 

#mh dom mon dow命令
0 0 * * * bash /usr/local/bin/backup.sh

该backup.sh脚本现在将在每天上午12:00执行。

需要 将backup.sh脚本复制到/ usr / local / bin /目录，以便正确执行此条目。该脚本可以驻留在文件系统的任何位置，只需适当更改脚本路径即可。

有关更深入的crontab选项，请参阅参考资料。

从档案馆恢复

创建存档后，测试存档很重要。可以通过列出它包含的文件来测试存档，但最好的测试是从存档中恢复文件。

  • 查看存档内容的列表。从终端提示符输入： 

    tar -tzvf /mnt/backup/host-Monday.tgz

  • 要将文件从存档还原到其他目录，请输入： 

    tar -xzvf /mnt/backup/host-Monday.tgz -C / tmp etc / hosts

    tar 的-C选项将提取的文件重定向到指定的目录。上面的示例将/ etc / hosts文件解压缩到/ tmp / etc / hosts。 tar 重新创建它包含的目录结构。

    另外，请注意前导“/”不在要还原的文件的路径上。

  • 要还原存档中的所有文件，请输入以下内容： 

    cd / 
sudo tar -xzvf /mnt/backup/host-Monday.tgz

这将覆盖当前文件系统上的文件。

    • dd：coreutils 包的一部分。一种低级实用程序，可以将数据从一种格式复制到另一种格式。

    • rsnapshot：用于创建整个文件系统副本的文件系统快照实用程序。

    • rsync：一种灵活的实用程序，用于创建文件的增量副本。
