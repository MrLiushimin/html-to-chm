Bacula的

Bacula是一个备份程序，使您可以跨网络备份，还原和验证数据。有针对Linux，Windows和Mac OS X的Bacula客户端 - 使其成为跨平台的网络解决方案。

概观

Bacula由几个组件和服务组成，用于管理备份和备份位置的文件：

  • Bacula Director：一种控制所有备份，恢复，验证和归档操作的服务。

  • Bacula控制台：允许与Director通信的应用程序。控制台有三个版本：

    • 基于文本的命令行版本。

    • 基于Gnome的GTK +图形用户界面（GUI）界面。

    • wxWidgets GUI界面。

  • Bacula File：也称为Bacula Client程序。此应用程序安装在要备份的计算机上，并负责Director请求的数据。

  • Bacula Storage：执行数据存储和恢复到物理介质的程序。

  • Bacula目录：负责维护备份的所有文件的文件索引和卷数据库，从而实现快速定位和恢复存档文件。Catalog支持三种不同的数据库MySQL，PostgreSQL和SQLite。

  • Bacula Monitor：允许监视Director，File守护进程和Storage守护进程。目前，Monitor仅作为GTK + GUI应用程序提供。

这些服务和应用程序可以在多个服务器和客户端上运行，也可以在备份单个磁盘或卷时安装在一台计算机上。

安装

如果使用MySQL或PostgreSQL作为数据库，您应该已经拥有了可用的服务。 Bacula不会为您安装它们。

有多个包含不同的Bacula组件。要安装Bacula，请在终端提示符下输入： 

sudo apt安装bacula

默认情况下，安装bacula包将使用 MySQL数据库作为目录。如果要使用SQLite或PostgreSQL，对于Catalog，分别安装bacula-director-sqlite3 或bacula-director-pgsql。

在安装过程中，将要求您为数据库管理员和bacula 数据库所有者提供凭据。数据库管理员需要具有创建数据库的适当权限，有关详细信息，请参阅MySQL。

组态

Bacula配置文件的格式基于包含由“{}”括号包围的 指令的资源。每个Bacula组件在/ etc / bacula目录中都有一个单独的文件 。

各种Bacula组件必须相互授权。这是使用密码指令完成的 。例如，/ etc / bacula / server_dir.conf文件中的存储资源密码 必须与/etc/bacula/bacula-sd.conf中的Director资源密码 匹配。

默认情况下，名为Client1的备份作业配置为归档Bacula目录。如果您计划使用服务器备份多个客户端，则应将此作业的名称更改为更具描述性的名称。要更改名称，请编辑/etc/bacula/bacula-dir.conf： 

＃
＃定义主要的夜间保存备份作业
＃默认情况下，此作业将备份到磁盘中 
工作{
  Name =“BackupServer”
  JobDefs =“DefaultJob”
  写Bootstrap =“/ var / lib / bacula / Client1.bsr”
}

上面的示例将作业名称更改为与计算机主机名匹配的BackupServer。将“BackupServer”替换 为适当的主机名或其他描述性名称。

该控制台可以用于查询主任关于就业，更要与使用控制台 非root用户，用户需要是在Bacula的组。要将用户添加到bacula组，请从终端输入以下内容： 

sudo adduser $ username bacula

用实际的用户名替换$ username。此外，如果要将当前用户添加到组中，则应该注销并重新登录以使新权限生效。

本地主机备份

本节介绍如何将单个主机上的指定目录备份到本地磁带驱动器。

  • 首先，需要配置存储设备。编辑/etc/bacula/bacula-sd.conf添加： 

    设备{
  名称=“磁带机”
  设备类型=磁带
  媒体类型= DDS-4
  存档设备= / dev / st0
  硬件结束介质=否;
  AutomaticMount =是; ＃设备打开时，阅读它
  AlwaysOpen =是;
  RemovableMedia =是的;
  RandomAccess = no;
  警报命令=“sh -c'bandinfo -f％c | grep TapeAlert'”
}

    该示例适用于DDS-4磁带机。调整“媒体类型”和“存档设备”以匹配您的硬件。

    您还可以取消注释文件中的其他示例之一。

  • 编辑/etc/bacula/bacula-sd.conf后，需要重新启动Storage守护程序： 

    sudo systemctl重新启动bacula-sd.service

  • 现在在/etc/bacula/bacula-dir.conf中添加一个存储资源以使用新设备： 

    ＃“磁带机”存储设备的定义
存储{
  Name = TapeDrive
  ＃不要在这里使用“localhost”    
  地址= backupserver＃NB在此处使用完全限定名称
  SDPort = 9103
  密码=“Cv70F6pf1t6pBopT4vQOnigDrR0v3LT3Cgkiyjc”
  设备=“磁带机”
  媒体类型=磁带
}

    该地址的指令必须是服务器的完全限定域名（FQDN）。将backupserver更改为实际主机名。

    另外，确保Password指令与/etc/bacula/bacula-sd.conf中的密码字符串匹配 。

  • 创建一个新的FileSet，它将通过添加以下内容来确定要备份的目录： 

    #localhostBacup FileSet。
FileSet {
  Name =“LocalhostFiles”
  包括{
    选项{
      签名= MD5
      压缩= GZIP
    }
    文件= /等
    文件= / home
  }
}

    此FileSet将备份/ etc和 / home目录。该选项资源配置指令文件集创建备份的每个文件的MD5签名，并使用GZIP压缩文件。

  • 接下来，为备份作业创建一个新的Schedule： 

    ＃LocalhostBackup Schedule  - 每日。
安排{
  Name =“LocalhostDaily”
  每天00:01运行=全天
}

    这项工作将在每天00:01或12:01运行。还有许多其他调度选项可用。

  • 最后创建工作： 

    #localhost备份。
工作{
  Name =“LocalhostBackup”
  JobDefs =“DefaultJob”
  启用=是
  等级=满
  FileSet =“LocalhostFiles”
  Schedule =“LocalhostDaily”
  存储= TapeDrive
  Write Bootstrap =“/var/lib/bacula/LocalhostBackup.bsr”
}

    该作业将每天对磁带驱动器执行完全备份。

  • 使用的每个磁带都需要有一个Label。如果当前磁带没有标签， Bacula将发送一封电子邮件通知您。要使用控制台标记磁带 ，请在终端中输入以下内容： 

    bconsole

  • 在Bacula Console提示符处输入： 

    标签

  • 然后，系统将提示您输入存储资源： 

    
自动选择目录：MyCatalog
使用目录“MyCatalog”
定义的存储资源是：
     1：文件
     2：TapeDrive
选择存储资源（1-2）：2

  • 输入新的卷名称： 

    
输入新的卷名：星期日
定义的池：
     1：默认
     2：划痕

    用所需标签替换星期日。

  • 现在，选择池： 

    
选择池（1-2）：1
在backupserver连接到存储后台程序TapeDrive：9103 ...
发送卷“星期日”插槽0的标签命令...

恭喜，您现在已将Bacula配置为将localhost备份到连接的磁带驱动器。
