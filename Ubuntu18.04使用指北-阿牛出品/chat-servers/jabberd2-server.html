Jabber即时消息服务器

Jabber是一种流行的即时消息协议，它基于XMPP，一种即时消息的开放标准，并被许多流行的应用程序使用。本节介绍如何在本地LAN上设置Jabberd 2服务器。该配置还可以适用于通过因特网向用户提供消息服务。

  • 安装
  • 组态
  • 参考

安装

要在终端中安装jabberd2，请输入： 

sudo apt install jabberd2

组态

将使用一些XML配置文件来配置jabberd2以进行 Berkeley DB用户身份验证。这是一种非常简单的身份验证形式。但是， jabberd2可以配置为使用LDAP，MySQL，PostgreSQL等进行用户身份验证。

首先，编辑/etc/jabberd2/sm.xml更改： 

  <ID> jabber.example.com </ ID>

将jabber.example.com替换为服务器的主机名或其他ID。

现在在<storage>部分中将<driver>更改为： 

   <驱动程序>分贝</驱动器>

接下来，编辑<local>部分中的/etc/jabberd2/c2s.xml更改： 

    <ID> jabber.example.com </ ID>

并在<authreg>部分中将<module>部分调整为： 

    <模块>分贝</模块>

最后，重启jabberd2以启用新设置： 

sudo systemctl restart jabberd2.service

您现在应该能够使用像Pidgin这样的Jabber客户端连接到服务器。

将Berkeley DB用于用户数据的优点是在配置后无需额外维护。如果您需要更多地控制用户帐户和凭据，建议使用其他身份验证方法。
