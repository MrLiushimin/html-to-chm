设置DM-Multipath概述

本节提供配置DM-Multipath的分步示例过程。它包括以下程序：

  • 基本DM-Multipath设置

  • 忽略本地磁盘

  • 在配置文件中添加更多设备

  • 设置DM-Multipath
  • 使用多路径支持进行安装
  • 生成多路径设备时忽略本地磁盘
  • 配置存储设备

设置DM-Multipath

在系统上设置DM-Multipath之前，请确保您的系统已更新并包含multipath-tools软件包。如果需要从SAN引导，则还需要multipath-tools-boot软件包。

基本/etc/multipath.conf中甚至不必存在，当multpath是没有伴随运行/etc/multipath.conf中，它吸引了来自它的内部数据库中找到一个合适的配置，它也吸引了来自它的内部黑名单。如果在没有配置文件的情况下运行multipath -ll之后，则不会发现多路径。必须继续增加详细程度以发现为什么没有创建多路径。考虑引用SAN供应商的文档，/ usr / share / doc / multipath-tools / examples中的多路径示例配置文件 以及实时多路径数据库：

#echo'show config'| multipathd -k> multipath.conf-live

要解决multipathd中的一个怪癖，当 /etc/multipath.conf中不存在，则上面的命令将返回任何结果，因为它是一个结果 合并之间 /etc/multipath.conf中，并在内存数据库。要解决此问题，请使用touch定义空的 /etc/multipath.conf，或者创建一个重新定义默认值的方法，如：

默认值{
        user_friendly_names没有
}
并重启multipathd：
#systemctl restart multipath-tools.service
现在“show config”命令将返回实时数据库。

使用多路径支持进行安装

为了使安装过程中多路径支持使用

install disk-detect / multipath / enable = true
在安装程序提示符下。如果找到多路径设备，这些设备将在安装期间显示为/ dev / mapper / mpath <X>。

配置存储设备

默认情况下，DM-Multipath支持支持DM-Multipath的最常见存储阵列。可以在multipath.conf.defaults文件中找到默认配置值，包括支持的设备 。

如果需要添加默认情况下不支持的存储设备作为已知多路径设备，请编辑/etc/multipath.conf文件并插入相应的设备信息。

例如，要添加有关HP Open-V系列的信息，条目如下所示，其中％n是设备名称：

设备 {
     设备{
            供应商“惠普”
            产品“OPEN-V。”
            getuid_callout“/ lib / udev / scsi_id --whitelisted --device = / dev /％n”
     }
}

有关配置文件的“设备”部分的更多信息，请参见“ 配置文件设备”一节。
