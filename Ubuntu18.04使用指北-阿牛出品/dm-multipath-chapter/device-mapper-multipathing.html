设备映射器多路径

设备映射器多路径（DM-Multipath）允许您将服务器节点和存储阵列之间的多个I / O路径配置到单个设备中。这些I / O路径是物理SAN连接，可包括单独的电缆，交换机和控制器。多路径聚合I / O路径，创建由聚合路径组成的新设备。本章概述了DM-Multipath的特性，这些特性是Ubuntu Server 12.04初始版本的新增功能。接下来，本章提供DM Multipath及其组件的高级概述，以及DM-Multipath设置的概述。

从0.4.8迁移

优先级检查程序不再作为独立二进制文件运行，而是作为共享库运行。此功能的键值名称也略有变化。将名为prio_callout的属性复制到prio，同时将参数修改为优先级检查器的名称，不再需要系统路径。转换示例：

设备{
        供应商“NEC”
        产品“DISK ARRAY”
        prio_callout mpath_prio_alua / dev /％n
        prio alua
}

概观

DM-Multipath可用于提供：

  • 冗余 DM-Multipath可以在主动/被动配置中提供故障转移。在主动/被动配置中，I / O的任何时候只使用一半路径。如果I / O路径的任何元素（电缆，交换机或控制器）发生故障，DM-Multipath将切换到备用路径。

  • 改进的性能性能DM-Multipath可以配置为主动/主动模式，其中I / O以循环方式分布在路径上。在某些配置中，DM-Multipath可以检测I / O路径上的负载并动态地重新平衡负载。

存储阵列概述

默认情况下，DM-Multipath支持支持DM-Multipath的最常见存储阵列。支持的设备可以在multipath.conf.defaults文件中找到。如果存储阵列支持DM-Multipath，并且在此文件中未默认配置，则可能需要将它们添加到DM-Multipath配置文件multipath.conf中。有关DM-Multipath配置文件的信息，请参见章节，DM-Multipath配置文件。某些存储阵列需要特殊处理I / O错误和路径切换。这需要单独的硬件处理程序内核模块

零件 描述
dm_multipath内核模块 重新路由I / O并支持路径和路径组的故障转移。
multipath命令 列出并配置多路径 设备。通常使用/etc/rc.sysinit启动，只要添加了块设备，它也可以由udev程序启动，或者它可以由initramfs文件系统运行。

DM-Multipath设置概述

DM-Multipath包括适用于常见多路径配置的已编译默认设置。设置DM多路径通常是一个简单的过程。使用DM-Multipath配置系统的基本过程如下：

  1. 安装multipath-tools 和multipath-tools-boot 软件包

  3. 如有必要，编辑multipath.conf配置文件以修改默认值并保存更新的文件。

  4. 启动multipath守护程序

  5. 更新初始ramdisk
