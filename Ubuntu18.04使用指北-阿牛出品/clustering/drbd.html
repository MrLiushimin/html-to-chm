DRBD

分布式复制块设备（DRBD）镜像在多个主机之间阻止设备。复制对主机系统上的其他应用程序是透明的。可以镜像任何块设备硬盘，分区，RAID设备，逻辑卷等。

要开始使用drbd，请先安装必要的软件包。从终端输入： 

sudo apt install drbd8-utils

如果您将虚拟内核用作虚拟机的一部分，则需要手动编译drbd模块。 在虚拟机中安装linux-server软件包可能更容易。

本节介绍如何设置drbd来复制单独的/ srv 分区，以及两个主机之间的ext3文件系统。分区大小不是特别相关，但两个分区的大小必须相同。

组态

此示例中的两个主机将被称为drbd01和drbd02。他们需要通过DNS或/ etc / hosts 文件配置名称解析。有关详细信息，请参阅域名服务（DNS）。

  • 要配置drbd，请在第一个主机上编辑/etc/drbd.conf： 

    全球{usage-count no; }
common {syncer {rate 100M; }}
资源r0 {
        协议C;
        启动 {
                wfc-timeout 15;
                degr-wfc-timeout 60;
        }
        网{
                cram-hmac-alg sha1;
                共同秘密的“秘密”;
        }
        在drbd01上{
                device / dev / drbd0;
                disk / dev / sdb1;
                地址192.168.0.1:7788;
                元磁盘内部;
        }
        在drbd02上{
                device / dev / drbd0;
                disk / dev / sdb1;
                地址192.168.0.2:7788;
                元磁盘内部;
        }
}

    /etc/drbd.conf中 还有许多其他选项，但对于此示例，它们的默认值很好。

  • 现在将/etc/drbd.conf复制到第二个主机： 

    scp /etc/drbd.conf drbd02：〜

  • 并且，在drbd02上将文件移动到/ etc： 

    sudo mv drbd.conf / etc /

  • 现在使用drbdadm实用程序初始化元数据存储。在每个服务器上执行： 

    sudo drbdadm create-md r0

  • 接下来，在两台主机上，启动drbd守护程序： 

    sudo systemctl start drbd.service

  • 在drbd01上，或您希望成为主要主机的主机上，输入以下内容： 

    sudo drbdadm  -  --overwrite-of-peer primary all

  • 执行上述命令后，数据将开始与辅助主机同步。要观察进度，请在 drbd02上输入以下内容： 

    看-n1 cat / proc / drbd

    要停止观看输出，请按Ctrl + c。

  • 最后，将文件系统添加到/ dev / drbd0并挂载它： 

    sudo mkfs.ext3 / dev / drbd0 
sudo mount / dev / drbd0 / srv

测试

要测试数据是否实际在主机之间同步，请将drbd01上的一些文件（主要文件）复制到/ srv： 

sudo cp -r / etc / default / srv

接下来，unmount / srv： 

sudo umount / srv

将主服务器降级为辅助角色： 

sudo drbdadm secondary r0

现在，在辅助服务器上将其提升为主要角色： 

sudo drbdadm primary r0

最后，挂载分区： 

sudo mount / dev / drbd0 / srv

使用ls，您应该看到/ srv / default从前主要 主机drbd01复制。
