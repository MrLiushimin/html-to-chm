QEMU

Qemu是一个机器模拟器，可以在不同的机器上运行一台机器的操作系统和程序。大多数情况下，它不是用作模拟器，而是用作与KVM或XEN内核组件协作的虚拟器。在这种情况下，它利用硬件的虚拟化技术来虚拟客户。

虽然qemu有一个命令行界面和一个监视器来与正在运行的guest 虚拟机进行交互，但很少有这种方式用于其他方式而不是开发目的。 Libvirt提供了特定版本和虚拟机管理程序的抽象，并封装了一些变通方法和最佳实践。

升级机器类型

这也是在Ubuntu Wiki上的一些限制和考虑因素中记录的

您可能希望将现有已定义guest虚拟机的计算机类型更新为：

  • 获取最新的安全修复程序和功能

  • 继续使用在现在不受支持的版本上创建的来宾

通常，建议在将qemu / kvm升级到新的主要版本时更新机器类型。但这可能永远不会是一个自动化任务，因为此更改是客户可见的。客户设备可能会改变外观，新功能将向客人公布等等。Linux通常非常擅长容忍这些更改，但它在很大程度上取决于guest虚拟机的设置和工作负载，而这必须由系统的所有者/管理员进行评估。已知通过更改硬件经常会产生严重影响的其他操作系统。考虑类似于将物理机的所有设备和固件替换为最新版本的机器类型更改 - 适用于此的所有注意事项也适用于评估机器类型升级。

与通常的主要配置更改一样，最好备份guest虚拟机定义和磁盘状态，以便能够进行回滚以防万一。没有集成的单个命令来通过virsh或类似工具更新机器类型。这是机器定义的正​​常部分。因此更新方式与大多数其他方式相同。

首先关闭机器并等待它达到该状态。

virsh shutdown <yourmachine>
#wait
virsh list --inactive
＃现在应该将您的机器列为“关闭”

然后编辑机器定义并在机器属性的类型标签中找到类型。

virsh编辑<yourmachine>
<type arch ='x86_64'machine ='pc-i440fx-xenial'> hvm </ type>

将其更改为您想要的值。如果您需要通过“-M”检查可用的类型？请注意，虽然提供上游类型作为方便，但仅支持Ubuntu类型。在那里你还可以看到当前的默认值。一般情况下，强烈建议您在可能的情况下更改为较新的类型以利用较新的功能，但也可以使用仅适用于较新设备虚拟化的错误修正。

kvm -M？
＃列出机器类型，例如
pc-i440fx-xenial Ubuntu 16.04 PC（i440FX + PIIX，1996）（默认）
...

在此之后，您可以再次启动您的客人。您可以根据需要从来宾和主机检查当前的机器类型。

virsh start <yourmachine>
＃来自主机，通过转储活动的xml定义来检查
virsh dumpxml <yourmachine> | xmllint --xpath“string（// domain / os / type / @ machine）” - 
＃或来自客人的dmidecode（如果支持）
sudo dmidecode | grep产品-A 1
        产品名称：标准PC（i440FX + PIIX，1996）
        版本：pc-i440fx-xenial

如果你像xml文件那样保持非实时定义，那么记住也要更新它们。
