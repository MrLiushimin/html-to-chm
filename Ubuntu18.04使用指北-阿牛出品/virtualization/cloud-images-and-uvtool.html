云图像和uvtool

  • 介绍
  • 使用uvtool创建虚拟机
  • 资源

介绍

由于Ubuntu是许多云平台上最常用的操作系统之一，稳定和安全的云图像的可用性变得非常重要。截至12.04，云基础设施之外的云图像的利用率已得到改善。现在可以使用这些映像创建虚拟机而无需完整安装。

使用uvtool创建虚拟机

从14.04 LTS开始，一个名为uvtool的工具极大地促进了使用云图像生成虚拟机（VM）的任务。 uvtool提供了一种简单的机制，可以在本地同步云图像，并在几分钟内使用它们创建新的VM。

Uvtool包

为了使用uvtool，将需要以下包及其依赖项：

  • uvtool

  • uvtool-libvirt的

要安装uvtool，请运行：

$ sudo apt -y install uvtool

这将安装uvtool的主要命令：

  • UVT-simplestreams，libvirt的

  • UVT-KVM

使用uvt-simplestreams-libvirt获取Ubuntu Cloud Image

这是uvtool带来的主要简化之一。它知道在哪里可以找到云图像，因此只需要一个命令即可获得新的云图像。例如，如果要同步amd64体系结构的所有云映像，则uvtool命令将为：

$ uvt-simplestreams-libvirt sync arch = amd64

在从Internet下载所有图像所需的时间之后，您将在本地存储一组完整的云图像。要查看已下载的内容，请使用以下命令：

如果您只想同步一个特定的云图像，则需要使用release =和arch =过滤器来识别需要同步的图像。

$ uvt-simplestreams-libvirt sync release = bionic arch = amd64

使用uvt-kvm创建VM

为了在创建虚拟机后连接到虚拟机，您必须具有可用于Ubuntu用户的有效SSH密钥。如果您的环境没有SSH密钥，则可以使用以下命令轻松创建一个：

要使用uvtool创建新虚拟机，请在终端中运行以下命令：

$ uvt-kvm创建第一个

这将使用本地可用的当前LTS云图像创建名为firsttest的VM 。如果要指定用于创建VM的版本，则需要使用release = filter：

$ uvt-kvm创建secondtest release = bionic

uvt-kvm等待可以用来等待VM的创建完成：

$ uvt-kvm等secondttest --insecure
警告：安全等待启动完成尚未实现; 使用--insecure。

连接到正在运行的VM

完成虚拟机创建后，您可以使用SSH连接到它： 

$ uvt-kvm ssh secondtest --insecure

目前，需要使用--insecure，因此只有在完全信任您的网络基础架构时才使用此机制连接到VM。

您还可以使用VM的IP地址使用常规SSH会话连接到VM。可以使用以下命令查询地址：

获取正在运行的VM列表

您可以使用以下命令获取系统上运行的VM列表：

$ uvt-kvm列表
secondtest

销毁你的VM

完成VM后，可以使用以下命令销毁它：

$ uvt-kvm摧毁第二次测试

更多uvt-kvm选项

以下选项可用于更改您正在创建的VM的某些特征：

  • --memory：RAM的数量，以兆字节为单位。默认值：512。

  • --disk：操作系统磁盘的大小（以GB为单位）。默认值：8。

  • --cpu：CPU核心数。默认值：1。

其他一些参数会对cloud-init配置产生影响：

  • --password密码：允许使用Ubuntu帐户和提供的密码登录VM。

  • --run-script-once script_file：第一次启动时，在VM上以root身份运行script_file，但永远不会再次运行。

  • --packages package_list：在首次启动时安装package_list中指定的逗号分隔的包。

有关所有可用修饰符的完整说明，请参见uvt-kvm的联机帮助页。
