使用实时服务器安装程序安装

安装Ubuntu Server Edition的基本步骤与安装任何操作系统的步骤相同。与Desktop Edition不同，Server Edition不包含图形安装程序。Live Server安装程序使用基于文本的控制台界面，该界面在默认虚拟控制台上运行。界面可以完全由输入，上下箭头键驱动（偶尔输入一些）。

如果您在安装过程中的任何时候需要，可以切换到另一个控制台（通过按Ctrl-Alt-F <n>或Ctrl-Alt-Right）来访问shell。在安装开始之前，您可以使用“后退”按钮返回上一个屏幕并选择不同的选项。

  • 从Ubuntu网站下载相应的ISO文件。

  • 从包含ISO文件的介质（例如USB密钥）引导系统。

  • 在启动提示符下，系统会要求您选择一种语言。

  • 从主启动菜单中，还有一些安装Ubuntu Server Edition的附加选项。您可以安装基本的Ubuntu Server，检查安装介质是否有缺陷，检查系统的RAM，或从第一个硬盘启动。本节的其余部分将介绍基本的Ubuntu Server安装。

  • 启动到安装程序后，它会询问您使用哪种语言。

  • 接下来，安装过程首先询问您的键盘布局。您可以要求安装程序尝试自动检测它，也可以从列表中手动选择它。安装的后期阶段将要求您键入ASCII字符，因此如果您选择的布局不允许，则系统将提示您输入组合键以在布局和您选择的布局之间切换。默认击键是Alt + Shift。

  • 接下来，安装程序提供了将系统安装为vanilla Ubuntu服务器，MAAS裸机云架控制器或MAAS区域控制器的选择。如果您选择其中一个MAAS选项，系统会要求您提供一些详细信息。

  • 安装程序将网络配置为在每个网络接口上运行DHCP。如果这不足以访问互联网，则应手动配置至少一个界面。选择要配置的接口。

  • 如果只能通过您环境中的代理访问Ubuntu存档，则可以在下一个屏幕上输入。如果不需要，请将该字段留空。

  • 然后，您可以选择让安装程序使用整个磁盘或手动配置分区。您创建分区的第一个磁盘将被选为引导磁盘，并在其上创建一个额外的分区以包含引导加载程序; 您可以使用“选择为启动磁盘”按钮将启动分区移动到其他驱动器。

    从此屏幕开始移动后，将开始安装进度。无法移回此屏幕或上一屏幕，并且您已配置安装程序要使用的磁盘上的任何数据都将丢失。

  • 下一个屏幕配置系统的初始用户。您可以从Launchpad或Github导入SSH密钥，但仍需要设置密码，因为该用户将通过sudo实用程序具有root访问权限。

  • 最后一个屏幕显示安装程序的进度。安装完成后，系统将提示您重新启动新安装的系统。
