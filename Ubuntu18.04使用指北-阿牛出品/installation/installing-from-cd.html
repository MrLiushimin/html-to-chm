使用debian-installer进行安装

安装Ubuntu Server Edition的基本步骤与安装任何操作系统的步骤相同。与Desktop Edition不同，Server Edition不包含图形安装程序。debian-installer安装程序使用基于控制台菜单的进程。

  • 从 Ubuntu网站下载相应的ISO文件。

  • 从包含ISO文件的介质（例如USB密钥）引导系统。

  • 在启动提示符下，系统会要求您选择一种语言。

  • 从主启动菜单中，还有一些安装Ubuntu Server Edition的附加选项。您可以安装基本的Ubuntu Server，检查CD-ROM中的缺陷，检查系统的RAM，从第一个硬盘启动，或者救出损坏的系统。本节的其余部分将介绍基本的Ubuntu Server安装。

  • 安装程序询问它应该使用哪种语言。之后，系统会要求您选择您的位置。

  • 接下来，安装过程首先询问您的键盘布局。您可以要求安装程序尝试自动检测它，也可以从列表中手动选择它。

  • 然后，安装程序会发现您的硬件配置，并使用DHCP配置网络设置。如果您不希望在下一个屏幕上使用DHCP，请选择“返回”，然后您可以选择“手动配置网络”。

  • 接下来，安装程序会询问系统的主机名。

  • 设置了一个新用户; 此用户将通过sudo实用程序具有root访问权限。

  • 用户设置完成后，系统将询问您是否要加密 主目录。

  • 接下来，安装程序会询问系统的时区。

  • 然后，您可以从多个选项中进行选择以配置硬盘驱动器布局。之后会询问您要安装哪个磁盘。在重写分区表或根据磁盘布局设置LVM之前，您可能会收到确认提示。如果选择LVM，将要求您提供根逻辑卷的大小。对于高级磁盘选项看到高级安装。

  • 然后安装Ubuntu基本系统。

  • 安装过程的下一步是确定如何更新系统。有三种选择：

    • 无自动更新：这需要管理员登录计算机并手动安装更新。

    • 自动安装安全更新：这将安装 无人值守升级包，该包将在没有管理员干预的情况下安装安全更新。有关详细信息，请参阅自动更新。

    • 使用Landscape管理系统：Landscape是Canonical提供的一项付费服务​​，用于帮助管理您的Ubuntu计算机。有关详细信息，请参阅 Landscape网站。

  • 您现在可以选择安装或不安装多个包任务。请参阅 包任务以获取详细信 此外，还有一个选项可以启动 aptitude以选择要安装的特定包。有关更多信息，请参阅Aptitude。

  • 最后，重新启动之前的最后一步是将时钟设置为UTC。

如果在安装过程中的任何时候您对默认设置不满意，请在任何提示时使用“返回”功能进入详细的安装菜单，以便修改默认设置。

在安装过程中的某些时候，您可能需要阅读安装系统提供的帮助屏幕。为此，请按F1。

有关详细说明，请再次参阅“ Ubuntu安装指南”。

包任务

在Server Edition安装期间，您可以选择安装其他软件包。包根据它们提供的服务类型进行分组。

  • DNS服务器：选择BIND DNS服务器及其文档。

  • LAMP服务器：选择现成的Linux / Apache / MySQL / PHP服务器。

  • 邮件服务器：此任务选择对通用邮件服务器系统有用的各种包。

  • OpenSSH服务器：选择OpenSSH服务器所需的包。

  • PostgreSQL数据库：此任务选择PostgreSQL数据库的客户端和服务器包。

  • 打印服务器：此任务将您的系统设置为打印服务器。

  • Samba文件服务器：此任务将您的系统设置为Samba文件服务器，尤其适用于具有Windows和Linux系统的网络。

  • Tomcat Java服务器：安装Apache Tomcat并需要依赖项。

  • 虚拟机主机：包括运行KVM虚拟机所需的软件包。

  • 手动选择包：执行aptitude，允许您单独选择包。

使用tasksel实用程序完成安装软件包组。Ubuntu（或Debian）与其他GNU / Linux发行版之间的一个重要区别是，在安装时，程序包也配置为合理的默认值，最终会提示您输入其他所需信息。同样，在安装任务时，不仅安装了软件包，还配置了软件包以提供完全集成的服务。

安装过程完成后，您可以通过在终端提示符下输入以下内容来查看可用任务列表： 

tasksel --list-tasks

输出将列出来自其他基于Ubuntu的发行版的任务，例如Kubuntu和Edubuntu。请注意，您还可以单独调用tasksel命令，这将显示可用的不同任务的菜单。

您可以使用--task-packages选项查看每个任务安装的软件包列表。例如，要列出使用DNS服务器任务安装的软件包，请输入以下内容： 

tasksel --task-packages dns-server

该命令的输出应列出： 

BIND9-DOC 
bind9utils 
BIND9

如果您在安装过程中没有安装其中一个任务，但是例如您决定将新的LAMP服务器设置为DNS服务器，只需从终端插入安装介质： 

sudo tasksel install dns-server
