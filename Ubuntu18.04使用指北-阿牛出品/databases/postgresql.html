PostgreSQL的

PostgreSQL是一个对象 - 关系数据库系统，具有传统商业数据库系统的功能，可在下一代DBMS系统中找到增强功能。

安装

要安装PostgreSQL，请在命令提示符中运行以下命令： 

sudo apt install postgresql

安装完成后，您应该根据需要配置 PostgreSQL服务器，尽管默认配置是可行的。

组态

PostgreSQL支持多种客户端身份验证方法。除非另行配置，否则IDENT身份验证方法用于postgres和本地用户。如果您想配置Kerberos等替代方案，请参阅 PostgreSQL管理员指南。

以下讨论假定您希望启用TCP / IP连接并使用MD5方法进行客户端身份验证。PostgreSQL配置文件存储在 / etc / postgresql / <version> / main 目录中。例如，如果安装PostgreSQL 9.5，配置文件将存储在 /etc/postgresql/9.5/main目录中。

要配置ident身份 验证，请在/etc/postgresql/9.5/main/pg_ident.conf 文件中添加条目。文件中有详细的评论以指导您。

要使其他计算机能够连接到PostgreSQL服务器，请编辑文件/etc/postgresql/9.5/main/postgresql.conf

找到#listen_addresses ='localhost'行并将其更改为： 

listen_addresses ='*'

要允许IPv4和IPv6连接，请将'localhost'替换为'::'

如果您知道自己在做什么，也可以编辑所有其他参数！有关详细信息，请参阅配置文件或PostgreSQL文档。

现在我们可以连接到PostgreSQL服务器，下一步是为postgres用户设置密码。在终端提示符处运行以下命令以连接到默认的PostgreSQL模板数据库： 

sudo -u postgres psql template1

上面的命令作为用户 postgres连接到PostgreSQL数据库template1。一旦连接到PostgreSQL服务器，您将处于SQL提示符下。您可以在psql 提示符下运行以下SQL命令来配置用户postgres的密码。 

ALTER USER postgres加密密码'your_password';

配置密码后，编辑文件 /etc/postgresql/9.5/main/pg_hba.conf以对postgres用户使用 MD5身份验证： 

本地所有postgres md5

最后，您应该重新启动PostgreSQL服务以初始化新配置。从终端提示符输入以下内容以重新启动PostgreSQL： 

sudo systemctl restart postgresql.service

上述配置无论如何都不完整。请参阅 PostgreSQL管理员指南以配置更多参数。

您可以使用PostgreSQL客户端测试来自其他计算机的服务器连接。 

sudo apt install postgresql-client 
psql -h postgres.example.com -U postgres -W

将域名替换为您的实际服务器域名。

备份

应定期备份PostgreSQL数据库。有关不同的方法，请参阅PostgreSQL管理员指南。

资源

  • 如上所述，PostgreSQL管理员指南 是一个很好的资源。该指南也可以在postgresql-doc-9.5包中找到。在终端中执行以下命令以安装软件包： 

    sudo apt install postgresql-doc-9.5

    要查看指南，请在浏览器的地址栏中输入file：///usr/share/doc/postgresql-doc-9.5/html/index.html。

  • 有关常规SQL信息，请参阅Rafe Colburn 使用SQL特别版。

  • 另外，有关更多信息，请参阅PostgreSQL Ubuntu Wiki页面。
