TCP / IP

传输控制协议和因特网协议（TCP / IP）是由国防高级研究计划局（DARPA）在20世纪70年代后期开发的标准协议集，作为不同类型的计算机和计算机网络之间的通信手段。TCP / IP是互联网的驱动力，因此它是地球上最受欢迎的一组网络协议。

TCP / IP简介

TCP / IP的两个协议组件涉及计算机网络的不同方面。 Internet协议，TCP / IP的“IP”是一种无连接协议，它仅使用IP数据报作为网络信息的基本单元处理网络数据包路由。IP数据报由标题后跟消息组成。的 传输控制协议是TCP / IP的“TCP”，使网络主机建立可用于交换的数据流的连接。TCP还保证连接之间的数据被传递，并且它以与从另一个网络主机发送的顺序相同的顺序到达一个网络主机。

TCP / IP配置

TCP / IP协议配置包含几个必须通过编辑相应的配置文件来设置的元素，或者部署诸如动态主机配置协议（DHCP）服务器之类的解决方案，而后者又可以配置为提供正确的TCP / IP配置自动设置网络客户端。必须正确设置这些配置值，以便于Ubuntu系统正常运行网络。

TCP / IP的常见配置元素及其用途如下：

  • IP地址 IP地址是一个唯一的标识字符串，表示为四个十进制数，范围从零（0）到二百五十五（255），以句点分隔，四个数字中的每一个代表八（8）位整个地址的总长度为32（32）位的地址。此格式称为 点分四位表示法。

  • 网络掩码子网掩码（或简称为 网络掩码）是一个本地位掩码或一组标志，它们将对网络有意义的IP地址部分与对子网有重要的位分开。例如，在C类网络中，标准网络掩码为255.255.255.0，它掩盖IP地址的前三个字节，并允许IP地址的最后一个字节保持可用于指定子网上的主机。

  • 网络地址网络地址表示构成IP地址的网络部分的字节。例如，A类网络中的主机12.128.1.2将使用12.0.0.0作为网络地址，其中十二（12）代表IP地址的第一个字节（网络部分），全部为零（0）剩下的三个字节代表潜在的主机值。使用专用IP地址192.168.1.100的网络主机将依次使用192.168.1.0的网络地址，该地址指定C类192.168.1网络的前三个字节，并为所有可能的主机指定零（0）。网络。

  • 广播地址广播地址是一个IP地址，它允许将网络数据同时发送到给定子网上的所有主机，而不是指定特定主机。IP网络的标准通用广播地址是255.255.255.255，但是该广播地址不能用于向因特网上的每个主机发送广播消息，因为路由器会阻止它。设置更合适的广播地址以匹配特定的子网。例如，在专用C类IP网络192.168.1.0上，广播地址为192.168.1.255。广播消息通常由诸如地址解析协议（ARP）和路由信息协议（RIP）之类的网络协议产生。

  • 网关地址网关地址是可以通过其访问网络上的特定网络或主机的IP地址。如果一个网络主机希望与另一个网络主机通信，并且该主机不在同一网络上，则 必须使用网关。在许多情况下，网关地址将是同一网络上的路由器的地址，后者又将流量传递到其他网络或主机，例如Internet主机。“网关地址”设置的值必须正确，否则您的系统将无法访问同一网络上的主机之外的任何主机。

  • 名称服务器地址名称服务器地址表示域名服务（DNS）系统的IP地址，它将网络主机名解析为IP地址。名称服务器地址有三个级别，可以按优先顺序指定： 主名称 服务器，辅助名称服务器和 第三级 域名服务器。为了使您的系统能够将网络主机名解析为相应的IP地址，您必须指定有权在系统的TCP / IP配置中使用的有效的名称服务器地址。在许多情况下，这些地址可以并且将由您的网络服务提供商提供，但许多免费且可公开访问的名称服务器可供使用，例如IP地址从4.2.2.1到4.2.2.6的Level3（Verizon）服务器。

IP路由

IP路由是指定和发现TCP / IP网络中可以发送网络数据的路径的手段。路由使用一组路由表 来指导网络数据包从其源到目的地的转发，通常通过许多称为路由器的中间网络节点。IP路由有两种主要形式：静态路由和 动态路由。

静态路由，涉及到手动添加IP路由系统的路由表，而这通常是由与操纵路由表进行路由命令。静态路由比动态路由具有许多优点，例如在较小网络上实现的简单性，可预测性（路由表总是预先计算，因此每次使用时路由完全相同），并且其他路由器的开销很低由于缺乏动态路由协议而导致网络链接。但是，静态路由确实也存在一些缺点。例如，静态路由仅限于小型网络，并且不能很好地扩展。由于路由的固定性质，静态路由也完全无法适应路由中的网络中断和故障。

动态路由依赖于具有从源到目的地的多条可能IP路由的大型网络，并利用特殊路由协议（例如路由器信息协议（RIP））来处理路由表中的自动调整，从而实现动态路由。与静态路由相比，动态路由具有多个优势，例如卓越的可扩展性以及适应网络路由故障和中断的能力。此外，路由表的手动配置较少，因为路由器相互学习它们的存在和可用路由。这种特性还消除了通过人为错误在路由表中引入错误的可能性。但是，动态路由并不完美

TCP和UDP

TCP是一种基于连接的协议，通过所谓的流控制提供错误纠正和有保证的数据传输。流控制确定何时需要停止数据流的流动，并且例如由于诸如冲突之类的问题应该重新发送先前发送的数据分组，从而确保完整和准确地传送数据。TCP通常用于交换重要信息，例如数据库事务。

另一方面，用户数据报协议（UDP）是一种无连接 协议，它很少处理重要数据的传输，因为它缺乏流量控制或任何其他方法来确保数据的可靠传送。UDP通常用于音频和视频流等应用中，由于缺少纠错和流量控制，UDP比TCP快得多，并且丢失少量数据包通常不会造成灾难性后果。

ICMP

Internet控制消息传递协议（ICMP）是对请求注释（RFC）＃792中定义的Internet协议（IP）的扩展，并支持包含控制，错误和信息性消息的网络数据包。ICMP由ping 实用程序等网络应用程序使用，可以确定网络主机或设备的可用性。ICMP返回的对网络主机和路由器等设备都有用的一些错误消息的示例包括 Destination Unreachable和Time Exceeded。

守护进程

守护进程是特殊的系统应用程序，通常在后台连续执行，并等待它们从其他应用程序提供的功能请求。许多守护进程都以网络为中心; 也就是说，在Ubuntu系统的后台执行的大量守护进程可能提供与网络相关的功能。这种网络守护进程的一些示例包括 提供Web服务器功能的超文本传输​​协议守护进程（httpd）; 所述SSH守护进程（sshd的），其提供安全远程登录壳和文件传输功能; 和Internet消息访问协议守护程序 （imapd），提供电子邮件服务。
