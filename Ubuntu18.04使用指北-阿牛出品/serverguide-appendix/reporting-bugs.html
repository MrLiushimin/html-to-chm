报告Ubuntu Server Edition中的错误

Ubuntu项目，以及Ubuntu Server，使用Launchpad 作为其bugtracker。要提交错误，您需要一个Launchpad帐户。 必要时在这里创建一个。

使用apport-cli报告错误

报告错误的首选方法是使用apport-cli命令。必须在受bug影响的计算机上调用它，因为它从运行它的系统收集信息并将其发布到Launchpad上的错误报告中。因此，如果系统没有运行桌面环境以便使用浏览器（与服务器通用）或者如果它没有Internet访问权限，那么将该信息提供给Launchpad可能是一个挑战。下面介绍了处理这些情况的步骤。

命令apport-cli和ubuntu-bug应该在CLI服务器上提供相同的结果。后者实际上是apport-bug的符号链接，它足够聪明，可以知道桌面环境是否正在使用，如果没有，将选择apport-cli。由于服务器系统倾向于仅使用CLI，因此本指南从一开始就选择了clort-cli。

Ubuntu中的错误报告需要针对特定​​的软件包提交，因此需要将受bug影响的包的名称（源包或程序名称/路径）提供给apport-cli： 

apport-cli PACKAGENAME

有关Ubuntu中的包的更多信息，请参阅包管理。

一旦apport-cli收集完信息，您就会被问到如何处理它。例如，报告vim中的错误： 

apport-cli vim

报告应用程序崩溃

该软件包提供所述的apport-CLI公用程序，Apport会，可以被配置为自动捕捉崩溃应用程序的状态。默认情况下启用此选项（在/ etc / default / apport中）。

应用程序崩溃后，如果启用，apport将在/ var / crash下存储崩溃报告： 

-rw-r ----- 1 peter whoopsie 150K 7月24日16:17 _usr_lib_x86_64-linux-gnu_libmenu-cache2_libexec_menu-cached.1000.crash

使用不带参数的apport-cli命令来处理任何挂起的崩溃报告。它将提供逐个报告。 

Apport会-CLI

***向开发人员发送问题报告？

问题报告发出后，请填写表格
自动打开网页浏览器。

你想干什么？你的选择是：
  S：发送报告（153.0 KB）
  V：查看报告
  K：保留报告文件以便稍后发送或复制到其他地方
  I：取消并忽略此程序版本的未来崩溃
  C：取消
请选择（S / V / K / I / C）：s

如果您发送报告，如上所述，将立即返回提示，然后 / var / crash目录将包含2个额外文件： 

-rw-r ----- 1 peter whoopsie 150K 7月24日16:17 _usr_lib_x86_64-linux-gnu_libmenu-cache2_libexec_menu-cached.1000.crash
-rw-rw-r-- 1 peter whoopsie 0 Jul 24 16:37 _usr_lib_x86_64-linux-gnu_libmenu-cache2_libexec_menu-cached.1000.upload
-rw ------- 1 whoopsie whoopsie 0 Jul 24 16:37 _usr_lib_x86_64-linux-gnu_libmenu-cache2_libexec_menu-cached.1000.uploaded

像这样发送崩溃报告不会立即导致创建新的公共错误。该报告将在Launchpad上设置为私有，这意味着它只对一组有限的bug分类器可见。然后，这些分类器将在创建公共错误之前扫描报告以查找可能的私有数据。
