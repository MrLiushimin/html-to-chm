CUPS - 打印服务器

Ubuntu打印和打印服务的主要机制是 通用UNIX打印系统（CUPS）。该打印系统是免费提供的便携式打印层，已成为大多数Linux发行版中打印的新标准。

CUPS管理打印作业和队列，并使用标准的Internet打印协议（IPP）提供网络打印，同时支持从点阵到激光以及介于两者之间的各种打印机。CUPS还支持PostScript打印机描述（PPD）和网络打印机的自动检测，并具有简单的基于Web的配置和管理工具。

安装

要在Ubuntu计算机上安装CUPS，只需在apt命令中使用sudo，并将包安装为第一个参数。完整的CUPS安装具有许多程序包依赖性，但它们可能都在同一命令行中指定。在终端提示符处输入以下内容以安装CUPS： 

sudo apt安装杯子

使用您的用户密码进行身份验证后，应该下载并安装这些软件包而不会出现错误。安装结束后，CUPS服务器将自动启动。

出于故障排除目的，您可以通过错误日志文件访问CUPS服务器错误：/ var / log / cups / error_log。如果错误日志没有显示足够的信息来解决您遇到的任何问题，可以通过将配置文件（下面讨论）中的LogLevel指令更改为“debug”甚至“debug2” 来增加CUPS日志的详细程度，一切，从默认的“信息”。如果进行此更改，请记住在解决问题后将其更改回来，以防止日志文件变得过大。

组态

通用UNIX打印系统服务器的行为是通过文件/etc/cups/cupsd.conf中包含的指令配置的。CUPS配置文件遵循与Apache HTTP服务器的主配置文件相同的语法，因此熟悉编辑Apache配置文件的用户在编辑CUPS配置文件时应该感到轻松。您最初可能希望更改的一些设置示例将在此处显示。

在编辑配置文件之前，您应该复制原始文件并保护其不被写入，因此您将原始设置作为参考，并在必要时重复使用。

复制/etc/cups/cupsd.conf文件并使用在终端提示符下发出的以下命令保护它不被写入： 

sudo cp /etc/cups/cupsd.conf /etc/cups/cupsd.conf.original 
sudo chmod aw /etc/cups/cupsd.conf.original

  • ServerAdmin：要配置CUPS服务器的指定管理员的电子邮件地址，只需使用首选文本编辑器编辑/etc/cups/cupsd.conf配置文件，然后相应地添加或修改ServerAdmin行。例如，如果您是CUPS服务器的管理员，并且您的电子邮件地址是“bjoy@somebigco.com”，那么您将修改ServerAdmin行以显示为： 

    ServerAdmin bjoy@somebigco.com

  • 侦听：默认情况下，在Ubuntu上，CUPS服务器安装仅侦听IP地址为127.0.0.1的环回接口 。为了指示CUPS服务器侦听实际网络适配器的IP地址，您必须通过添加Listen指令指定主机名，IP地址或可选的IP地址/端口配对。例如，如果您的CUPS服务器位于IP地址为192.168.10.250的本地网络上，并且您希望该子网上的其他系统可以访问它，则可以编辑 /etc/cups/cupsd.conf并添加一个Listen指令，如下： 

    听127.0.0.1:631＃现有的环回听
听/var/run/cups/cups.sock#现有套接字听
听192.168.10.250:631＃在LAN接口上侦听，端口631（IPP）

    在上面的示例中，如果您不希望cupsd 在该接口上侦听，则可以注释掉或删除对Loopback地址（127.0.0.1）的引用，但是宁愿它只在局域网的以太网接口上侦听（LAN）。要启用侦听绑定了某个主机名的所有网络接口（包括Loopback），您可以为主机名socrates创建一个Listen条目 ： 

    听socrates：631＃在所有界面上搜索主机名'socrates'

    或者省略Listen指令并改为使用Port，如：

    端口631＃在所有接口上侦听端口631

有关CUPS服务器配置文件中配置指令的更多示例，请通过在终端提示符处输入以下命令来查看关联的系统手册页： 

男人cupsd.conf

每当您对/etc/cups/cupsd.conf配置文件进行更改时，您都需要通过在终端提示符下键入以下命令来重新启动CUPS服务器： 

sudo systemctl restart cups.service

Web界面

可以使用Web界面配置和监视CUPS，默认情况下，该界面位于http：// localhost：631 / admin。Web界面可用于执行所有打印机管理任务。

要通过Web界面执行管理任务，您必须在服务器上启用root帐户，或者在lpadmin组中以用户身份进行身份验证。出于安全原因，CUPS不会对没有密码的用户进行身份验证。

要将用户添加到lpadmin组，请在终端提示符下运行： 

sudo usermod -aG lpadmin用户名

Web界面 的“ 文档/帮助”选项卡中提供了更多文档。
