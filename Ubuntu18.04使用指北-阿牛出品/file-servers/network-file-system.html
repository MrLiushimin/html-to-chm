网络文件系统（NFS）

NFS允许系统通过网络与其他人共享目录和文件。通过使用NFS，用户和程序可以访问远程系统上的文件，就像它们是本地文件一样。

NFS可以提供​​的一些最显着的好处是：

  • 本地工作站使用较少的磁盘空间，因为常用数据可以存储在一台计算机上，并且仍然可以通过网络访问其他人。

  • 用户不需要在每台网络机器上都有单独的主目录。可以在NFS服务器上设置主目录，并使其在整个网络中可用。

  • 网络上的其他计算机可以使用诸如软盘，CDROM驱动器和USB Thumb驱动器之类的存储设备。这可以减少整个网络中可移动介质驱动器的数量。

安装

在终端提示符处输入以下命令以安装NFS服务器： 

sudo apt安装nfs-kernel-server

组态

您可以通过将目录添加到/ etc / exports文件来配置要导出的目录。例如： 

/ ubuntu *（ro，sync，no_root_squash）
/ home *（rw，sync，no_root_squash）

您可以使用其中一种主机名格式替换*。使主机名声明尽可能具体，以便不需要的系统无法访问NFS挂载。

要启动NFS服务器，可以在终端提示符下运行以下命令： 

sudo systemctl start nfs-kernel-server.service

NFS客户端配置

使用mount命令从另一台计算机挂载共享NFS目录，方法是在终端提示符下键入与以下类似的命令行： 

sudo mount example.hostname.com:/ubuntu / local / ubuntu

必须存在 挂载点目录/ local / ubuntu。/ local / ubuntu目录中应该没有文件或子目录 。

从另一台计算机挂载NFS共享的另一种方法是在/ etc / fstab文件中添加一行。该行必须说明NFS服务器的主机名，正在导出的服务器上的目录以及要挂载NFS共享的本地计算机上的目录。

/ etc / fstab 文件中 的行的一般语法如下： 

example.hostname.com:/ubuntu / local / ubuntu nfs rsize = 8192，wsize = 8192，timeo = 14，intr

如果您在安装NFS共享时遇到问题，请确保在您的客户端上安装了nfs-common软件包。要安装 nfs-common，请在终端提示符处输入以下命令： 

sudo apt install nfs-common
