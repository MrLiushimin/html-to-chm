FTP服务器

文件传输协议（FTP）是用于在计算机之间下载文件的TCP协议。过去，它也被用于上传，但由于该方法不使用加密，因此用户凭据以及以明文形式传输的数据很容易被截获。所以，如果你在这里寻找一种方式来上传和安全地下载文件，请参见上一节的OpenSSH在远程管理来代替。

FTP适用于客户端/服务器模型。服务器组件称为FTP守护程序。它不断侦听来自远程客户端的FTP请求。收到请求后，它会管理登录并设置连接。在会话期间，它执行FTP客户端发送的任何命令。

可以通过两种方式管理对FTP服务器的访问：

  • 匿名

  • 经过身份验证

在匿名模式下，远程客户端可以使用名为“anonymous”或“ftp”的默认用户帐户访问FTP服务器，并发送电子邮件地址作为密码。在Authenticated模式下，用户必须拥有帐户和密码。后一种选择非常不安全，除特殊情况外不应使用。如果您希望安全地传输文件，请参阅OpenSSH-Server部分中的SFTP。用户对FTP服务器目录和文件的访问权限取决于为登录时使用的帐户定义的权限。作为一般规则，FTP守护程序将隐藏FTP服务器的根目录并将其更改为FTP主目录。

vsftpd - FTP服务器安装

vsftpd是Ubuntu中提供的FTP守护进程。它易于安装，设置和维护。要安装vsftpd， 您可以运行以下命令： 

sudo apt install vsftpd

匿名FTP配置

默认情况下，vsftpd的是没有配置为允许匿名下载。如果您希望通过更改以下命令启用匿名下载编辑/etc/vsftpd.conf： 

anonymous_enable =是

在安装过程中，将创建一个ftp用户，其主目录为/ srv / ftp。这是默认的FTP目录。

如果您希望更改此位置， 例如，在/ srv / files / ftp中，只需在另一个位置创建一个目录并更改ftp用户的主目录： 

sudo mkdir / srv / files / ftp 
sudo usermod -d / srv / files / ftp ftp

进行更改后重新启动vsftpd： 

sudo systemctl restart vsftpd.service

最后，如果您希望使用默认值，请将您希望通过匿名FTP提供的任何文件和目录复制到/ srv / files / ftp或/ srv / ftp。

用户验证的FTP配置

默认情况下，vsftpd配置为对系统用户进行身份验证并允许他们下载文件。如果您希望用户能够上传文件，请编辑 /etc/vsftpd.conf： 

WRITE_ENABLE = YES

现在重启vsftpd： 

sudo systemctl restart vsftpd.service

现在，当系统用户登录到FTP时，他们将从他们的主目录开始，在 那里他们可以下载，上传，创建目录等。

同样，默认情况下，不允许匿名用户将文件上载到FTP服务器。要更改此设置，您应取消注释以下行，然后重新启动vsftpd： 

anon_upload_enable = YES

启用匿名FTP上载可能会带来极大的安全风险。最好不要在直接从Internet访问的服务器上启用匿名上载。

配置文件包含许多配置参数。配置文件中提供了有关每个参数的信息。或者，您可以参考手册页man 5 vsftpd.conf以获取每个参数的详细信息。

保护FTP

/etc/vsftpd.conf中 有一些选项可以帮助使vsftpd更安全。例如，通过取消注释，可以将用户限制为其主目录： 

chroot_local_user = YES

您还可以将特定用户列表限制为其主目录： 

chroot_list_enable = YES
chroot_list_file =的/ etc / vsftpd.chroot_list

取消注释上述选项后，创建一个/etc/vsftpd.chroot_list， 其中包含每行一个用户列表。然后重启vsftpd： 

sudo systemctl restart vsftpd.service

此外，/ etc / ftpusers文件是不允许 FTP访问的用户列表。默认列表包括root，daemon，nobody等。要为其他用户禁用FTP访问，只需将它们添加到列表中即可。

FTP也可以使用FTPS加密。与SFTP不同， FTPS是基于安全套接字层（SSL）的FTP。 SFTP是通过加密SSH连接的类似FTP的会话。一个主要的区别是SFTP的用户需要在系统上拥有一个shell帐户，而不是一个 nologin贝壳。为所有用户提供shell可能不适用于某些环境，例如共享Web主机。但是，可以将此类帐户限制为仅SFTP并禁用shell交互。有关更多信息，请参阅OpenSSH-Server部分。

要配置FTPS，请编辑/etc/vsftpd.conf并在底部添加： 

ssl_enable =是

另外，请注意证书和密钥相关选项： 

rsa_cert_file =的/ etc / SSL /证书/ SSL-CERT-snakeoil.pem
rsa_private_key_file =的/ etc / SSL /私人/ SSL-CERT-snakeoil.key

默认情况下，这些选项设置为ssl-cert 包提供的证书和密钥。在生产环境中，应将这些替换为为特定主机生成的证书和密钥。有关证书的更多信息，请参阅证书。

现在重启vsftpd，非匿名用户将被迫使用 FTPS： 

sudo systemctl restart vsftpd.service

要允许具有/ usr / sbin / nologin shell的用户访问FTP，但没有shell访问权限，请编辑/ etc / shells添加nologin shell： 

＃/ etc / shells：有效的登录shell
/斌/ CSH
/ bin / sh的
在/ usr / bin中/ ES
在/ usr / bin中/ KSH
/斌/ KSH
在/ usr / bin中/ RC
在/ usr / bin中/ tcsh的
/斌/ tcsh的
在/ usr / bin中/ ESH
/斌/破折号
/斌/庆典
/斌/ rbash
在/ usr / bin中/屏幕
/ usr / sbin目录/ nologin的

这是必要的，因为默认情况下vsftpd使用PAM进行身份验证，而 /etc/pam.d/vsftpd配置文件包含： 

auth需要pam_shells.so

shell PAM模块限制对/ etc / shells 文件中列出 的shell的访问。

最流行的FTP客户端可以配置为使用FTPS进行连接。该lftp的命令行FTP客户端必须使用FTPS和能力。
