iSCSI启动器

iSCSI（Internet小型计算机系统接口）是一种允许通过网络传输SCSI命令的协议。通常，iSCSI在SAN（存储区域网络）中实现，以允许服务器访问大量硬盘空间。iSCSI协议将客户端称为启动器，将iSCSI服务器称为目标。

Ubuntu Server可以配置为iSCSI启动器和目标。本指南提供了设置iSCSI启动器的命令和配置选项。假设您已在本地网络上拥有iSCSI目标，并具有连接到它的适当权限。设置目标的说明在硬件提供商之间差异很大，因此请查阅供应商文档以配置特定的iSCSI目标。

iSCSI Initiator安装

要将Ubuntu Server配置为iSCSI启动器，请安装open-iscsi软件包。在终端输入： 

sudo apt install open-iscsi

iSCSI启动器配置

一旦开放的iSCSI安装包，编辑/etc/iscsi/iscsid.conf改变如下： 

node.startup =自动

您可以使用iscsiadm实用程序检查可用的目标。在终端中输入以下内容： 

sudo iscsiadm -m discovery -t st -p 192.168.0.10

  • -m：确定iscsiadm执行的模式。

  • -t：指定发现的类型。

  • -p：选项表示目标IP地址。

将示例192.168.0.10更改为网络上的目标IP地址。

如果目标可用，您应该看到类似于以下内容的输出： 


192.168.0.10:3260,1 iqn.1992-05.com.emc:sl7b92030000520000-2

上面的iqn编号和IP地址将根据您的硬件而有所不同。

您现在应该能够连接到iSCSI目标，并且根据您的目标设置，您可能必须输入用户凭据。登录iSCSI节点： 

sudo iscsiadm -m node --login

dmesg | grep sd

在上面的输出中，sdb是新的iSCSI磁盘。记住这只是一个例子; 您在屏幕上看到的输出会有所不同。

接下来，创建分区，格式化文件系统并装入新的iSCSI磁盘。在终端输入： 

sudo fdisk / dev / sdb 
n 
p 
输入
w

上面的命令来自fdisk实用程序; 有关详细说明，请参阅man fdisk。此外，cfdisk实用程序有时更加用户友好。

现在格式化文件系统并将其挂载到/ srv作为示例： 

sudo mkfs.ext4 / dev / sdb1 
sudo mount / dev / sdb1 / srv

最后，在/ etc / fstab中添加一个条目以在引导期间安装iSCSI驱动器： 

/ dev / sdb1 / srv ext4默认值，auto，_ netdev 0 0

通过重新启动服务器确保一切正常运行是个好主意。
