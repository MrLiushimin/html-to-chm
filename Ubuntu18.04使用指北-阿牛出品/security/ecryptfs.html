eCryptfs

eCryptfs是一个符合POSIX标准的企业级堆叠加密文件系统，适用于Linux。分层在文件系统层之上eCryptfs保护文件，无论底层文件系统，分区类型等。

在安装过程中，有一个加密/ home分区的选项。这将自动配置加密和挂载分区所需的所有内容。

例如，本节将介绍使用eCryptfs加密/ srv的配置。

使用eCryptfs

首先，安装必要的包。从终端提示输入： 

sudo apt install ecryptfs-utils

现在挂载要加密的分区： 

sudo mount -t ecryptfs / srv / srv

然后，系统将提示您提供有关ecryptfs应如何加密数据的一些详细信息。

要测试放在/ srv中的文件是否确实加密，请将/ etc / default 文件夹复制到/ srv： 

sudo cp -r / etc / default / srv

现在卸载/ srv，并尝试查看文件： 

sudo umount / srv 
cat / srv / default / cron

使用ecryptfs 重新安装/ srv将使数据再次可见。

自动挂载加密分区

有几种方法可以在启动时自动挂载ecryptfs加密文件系统。此示例将使用包含挂载选项的/root/.ecryptfsrc文件以及驻留在USB密钥上的密码短语文件。

首先，创建包含以下内容的/root/.ecryptfsrc： 

键=密码：passphrase_passwd_file = / MNT / USB / passwd_file.txt
ecryptfs_sig = 5826dd62cf81c615
ecryptfs_cipher = AES
ecryptfs_key_bytes = 16
ecryptfs_passthrough =正
ecryptfs_enable_filename_crypto =正

将ecryptfs_sig调整为/root/.ecryptfs/sig-cache.txt中的签名。

接下来，创建/mnt/usb/passwd_file.txt密码文件： 

passphrase_passwd = [秘密]

现在将必要的行添加到/ etc / fstab： 

/ dev / sdb1 / mnt / usb ext3 ro 0 0
/ srv / srv ecryptfs默认为0 0

确保在加密分区之前安装了USB驱动器。

最后，重启并使用eCryptfs挂载/ srv。

其他公用事业

的ecryptfs-utils的封装件包括几个其他有用的工具：

  • ecryptfs-setup-private：创建一个〜/ Private目录以包含加密信息。此实用程序可由非特权用户运行，以使数据与系统上的其他用户保持私密。

  • ecryptfs-mount-private和ecryptfs-umount-private将挂载和卸载用户的〜/ Private目录。

  • ecryptfs-add-passphrase：为内核密钥环添加一个新密码。

  • ecryptfs-manager：管理eCryptfs对象，如密钥。

  • ecryptfs-stat：允许您查看文件的ecryptfs元信息。
